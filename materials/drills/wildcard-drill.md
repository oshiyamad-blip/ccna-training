# 計算ドリル: ワイルドカードマスク

> 配置先: ドキュメント `00_ガイダンス > 計算ドリル`
> 想定運用: Day12（OSPF）・Day17（ACL）前日の集中ドリル、またはWeek0以降の毎朝の
> 振り返りで5問

## この教材の使い方

Day3・Day12・Day17 のシミュレーションで、2進数からサブネット計算・ワイルドカード
マスクへと進む計算が繰り返しつまずきの原因になっています。`binary-drill.md` で
2進数・16進数の土台を固めたうえで、このドリルではその先の「ワイルドカードマスク」
だけを反復練習します。

- 全 45 問を、レベル別に 15 問ずつ収録しています（初級・中級・上級）。
- 毎朝の振り返りで、通し番号順に 5 問ずつ解いてください。1 レベルが 3 日分
  （15 問 ÷ 5 問）にあたります。全問を終えるまでおよそ 9 営業日です。
- Day12・Day17 の前日は、該当レベル（初級〜上級のうち苦手な範囲、または
  上級 Q31〜45）をまとめて解く「集中ドリル」として使ってください。
- 解答はすべて末尾にまとめてあります。自己採点し、間違えた問題番号は
  `templates/error-log.md` に記録してください。
- 電卓・PC の計算機能は使わず、手計算で解いてください。

## おさらい: ワイルドカードマスクとサブネットマスクの関係

詳しい解説は本編講義（Week0・Day12・Day17）で扱っています。ここでは反復練習に
必要な最小限だけを再掲します。

**基本の関係式**

```
ワイルドカードマスク = 255.255.255.255 - サブネットマスク
```

オクテットごとに `255` からサブネットマスクの値を引くだけで求まります。逆に、
ワイルドカードマスクからサブネットマスクを求めるときも、同じ式をオクテットごとに
適用します（引き算は可換なので同じ計算で往復できます）。

**範囲の読み取り**

ネットワークアドレスとワイルドカードマスクが分かれば、マッチする範囲は次の式で
求まります。

```
先頭アドレス = ネットワークアドレス
末尾アドレス = ネットワークアドレス + ワイルドカードマスク（オクテットごとに加算）
```

ワイルドカードマスクの値が `n` なら、マッチするアドレスの個数は `n + 1` 個です
（例: `0.0.0.31` なら 32 個）。ネットワークアドレスは、その個数の倍数になっている
必要があります（例: 32 個ブロックなら、末尾オクテットが 32 の倍数）。倍数になって
いない場合、問題として不成立（範囲がずれる）なので注意してください。

**検算方法**

- サブネットマスク⇔ワイルドカードマスクを求めたら、もう一度逆方向に計算して
  元の値に戻るか確認する。
- 範囲を求めたら、末尾アドレスからネットワークアドレスを引き算し、ワイルドカード
  マスクの値と一致するか確認する。
- OSPF の `network` 文・ACL の行を書いたら、ワイルドカードマスクの箇所だけを
  抜き出して「サブネットマスクではなくワイルドカードマスクになっているか」を
  必ず見直す（最も多いミスは、サブネットマスクをそのまま書いてしまうこと）。
