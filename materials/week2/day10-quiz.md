## 解答・解説（翌日公開・講師用）

| 問 | 解答 | 解説 |
|---|---|---|
| Q1 | A | VLAN はブロードキャストドメインを分割し、不要なブロードキャストトラフィックの影響範囲を限定する目的で導入する |
| Q2 | C | アクセスポート化には `switchport mode access` と `switchport access vlan <番号>` の 2 つが必要 |
| Q3 | D | 音声 VLAN は IP 電話のトラフィックをデータ VLAN と分離し、QoS（優先制御）を適用しやすくするための仕組み |
| Q4 | B | `show vlan brief` の Ports 欄にはアクセスポートのみが VLAN 単位で列挙され、トランクポートは一覧に表示されない |
| Q5 | A | 802.1Q タグは 4 バイト（TPID 2 バイト＋ TCI 2 バイト、うち VLAN ID は 12 ビット）で構成される |
| Q6 | C | ネイティブ VLAN が両端で不一致だと、タグなしフレーム（CDP・STP 等の制御トラフィックを含む）が誤った VLAN として扱われ、警告が出力される |
| Q7 | D | `switchport trunk allowed vlan <VLAN リスト>` で、トランクを通過させる VLAN を制限できる |
| Q8 | B | `switchport nonegotiate` は DTP のネゴシエーションフレームの送受信自体を停止する |
| Q9 | A | `encapsulation dot1Q <VLAN 番号>` でサブインタフェースにカプセル化方式と対応 VLAN を指定する |
| Q10 | C | L3 スイッチは既定でルーティング機能が無効になっており、`ip routing` で有効化しないと SVI 間の転送ができない |
| Q11 | D | SVI（Switched Virtual Interface）は L3 スイッチ上に VLAN ごとの仮想インタフェースを作成し IP アドレスを割り当てる方式 |
| Q12 | B | プライオリティが同値の場合、次の比較基準は MAC アドレスであり、最小の MAC アドレスを持つスイッチが選ばれる |
| Q13 | A | ルートポートは各非ルートスイッチにつき必ず 1 つだけ選出される、ルートブリッジへの最短経路のポート |
| Q14 | C | RSTP はプロポーザル / アグリーメントのハンドシェイクにより、802.1D の最大約 50 秒に対して数秒程度で収束する |
| Q15 | D | BPDU Guard は、エッジポート（PortFast 設定ポート）が BPDU を受信した際にそのポートを err-disable にして無効化する |
| Q16 | B | LACP は active-active、active-passive の組み合わせで成立する。passive-passive のみ不成立 |
| Q17 | A | 無線は送信中に自局の送信波が支配的になり、他局からの電波を同時に受信して衝突を検出することが技術的に困難。そのため事前にチャネルの空きを確認し衝突を未然に防ぐ CSMA/CA を用いる |
| Q18 | A | 2.4GHz 帯で互いに重ならないチャネルは 1・6・11 の 3 つのみ |
| Q19 | C | 5GHz 帯はチャネル数が多く高速だが到達距離が短い。6GHz 帯は Wi-Fi 6E（802.11ax の拡張）で追加された |
| Q20 | D | CAPWAP の制御トンネルは UDP 5246、データトンネルは UDP 5247 を使用する |
| Q21 | A | Split MAC では、ビーコン送出・ACK 応答・暗号化 / 復号などリアルタイム性が求められる処理を AP が担う。認証やアソシエーション管理などの管理処理は WLC が担う |
| Q22 | B | join の失敗は SSID やパスフレーズではなく CAPWAP 確立段階の問題であることが多い。DHCP オプション 43 で通知した WLC 管理 IP が実際の値と食い違うと、LAP が WLC を発見できず Registered にならない |
| Q23 | B | FlexConnect は WAN 越しの拠点向けで、WLC との接続断時もローカルスイッチングで通信を継続できる |
| Q24 | A | WPA3-Personal は PSK の鍵交換を SAE に置き換え、オフライン辞書攻撃への耐性と前方秘匿性を向上させている |
| Q25 | C | dynamic インタフェースは、無線クライアントを収容する VLAN に対応するインタフェース種別 |
| Q26 | D | CDP は Cisco 独自のプロトコルで、既定のアドバタイズ間隔 60 秒・ホールドタイム 180 秒、Cisco 機器では既定で有効 |
| Q27 | B | LLDP は既定で無効なため `lldp run` で有効化する。既定の送信間隔は 30 秒、ホールドタイムは 120 秒 |
| Q28 | 例 | 「自律型 AP は台数が増えると SSID やセキュリティ設定を 1 台ずつ個別に管理する必要があり運用負荷が高い。Lightweight AP＋WLC 構成では、AP は CAPWAP（制御 UDP5246・データ UDP5247）で WLC とトンネルを確立し、Split MAC によりビーコン送出や暗号化などのリアルタイム処理は AP が、認証やポリシー適用などの管理処理は WLC が集中して担う。これにより多数の AP の設定・ポリシーを WLC 側でまとめて一元管理でき、ローミングや RF 最適化も行いやすくなる」という趣旨で、CAPWAP・Split MAC・集中管理のメリットに触れていれば正解 |

**採点基準**: 1 問 4 点（28 問 × 4 点 = 112 点満点）。70%未満（78 点未満）は
再テスト対象とする。Q28 は記述式のため、CAPWAP・Split MAC・集中管理という
主要な論点のいずれかに触れていれば部分点を含め満点として扱う。
