# Day 0: Packet Tracer を使えるようにしよう（環境構築）

> 配置先: ドキュメント `00_ガイダンス > Day00 環境構築`
> 所要時間の目安: 1.5〜2 時間 ／ **開講前日までに完了**してください
> 詰まったら: 課題 `[Day00] 環境構築` のコメントで遠慮なく質問を

この研修では、Cisco 公式の無料ネットワークシミュレータ **Cisco Packet Tracer** を
毎日使います。このページの手順どおりに進めれば、ネットワークの知識ゼロでも
約 2 時間で「機器を配置して操作できる」状態になります。

やることは 3 つだけです。

1. 無料アカウントを作る（15 分）
2. Packet Tracer をインストールする（30 分）
3. ミニ演習で動作確認する（30〜60 分）

---

## Step 1: Cisco NetAcad の無料アカウントを作る（15 分）

Packet Tracer のダウンロードには Cisco Networking Academy（NetAcad）の
無料アカウントが必要です。

1. ブラウザで <https://www.netacad.com/cisco-packet-tracer> を開く
2. 画面の案内から **Sign up（無料登録）** を選ぶ
3. メールアドレスを入力して登録（Google アカウントでのサインアップも可能）
4. 届いた確認メールのリンクをクリックして、アカウントを有効化する
5. ログインできることを確認する

> **つまずきポイント**
> - 確認メールが届かない → 迷惑メールフォルダを確認。会社アドレスで届かない場合は
>   個人アドレスでの登録も可
> - 画面は英語ですが、登録に必要な入力はメールアドレスと氏名程度です

> **注意**: インストーラを他の人からコピーしてもらう配布はライセンス違反です。
> 必ず自分のアカウントでダウンロードしてください。

## Step 2: Packet Tracer をインストールする（30 分）

1. ログインした状態で Resource Hub
   （<https://www.netacad.com/resources/lab-downloads>）を開く
2. **Cisco Packet Tracer** の欄から、自分の PC に合うインストーラをダウンロード
   - Windows: `64 bit .exe`
   - macOS: `.dmg`（Apple Silicon / Intel 共通）
   - Linux: `.deb`（Ubuntu 向け）
3. ダウンロードしたファイルを実行し、**すべて既定設定のまま**インストール
4. インストール完了後、Packet Tracer を起動する
5. 起動時にログイン画面が出たら、Step 1 のアカウントでログインする
   - **Keep me logged in** にチェックを入れると次回からログイン省略できます

> **つまずきポイント**
> - Windows で「WindowsによってPCが保護されました」→「詳細情報」→「実行」
> - macOS で「開発元を確認できません」→ システム設定 > プライバシーとセキュリティ
>   から許可
> - 会社支給 PC でインストールできない → 管理者権限が必要です。情シスへ申請を

## Step 3: ミニ演習で動作確認する（30〜60 分）

インストールできたかの確認を兼ねて、**研修で毎日行う操作**を一通り体験します。
ここでは意味が分からなくて大丈夫です。「触れる」ことがゴールです。

### 3-1. 機器を置いてつなぐ

1. Packet Tracer を起動し、新規ファイルを開く
2. 画面左下のデバイスボックスから **[End Devices] → PC** を
   中央の白い領域（ワークスペース）へ **2 台**ドラッグして置く
3. **[Network Devices] → [Switches] → 2960** を 1 台置く
4. 稲妻マークの**接続アイコン**をクリック →
   **Copper Straight-Through（実線のケーブル）** を選ぶ
5. PC0 をクリック → `FastEthernet0` を選ぶ → スイッチをクリック →
   `FastEthernet0/1` を選ぶ
6. 同様に PC1 をスイッチの `FastEthernet0/2` へつなぐ
7. ケーブル両端の丸印が**緑色**になるまで待つ（30 秒ほどかかります。
   橙色のうちは準備中です）

### 3-2. PC に住所（IP アドレス）を設定する

1. PC0 をクリック → **[Desktop] タブ → IP Configuration** を開く
2. 次のとおり入力する
   - IP Address: `192.168.1.10`
   - Subnet Mask: `255.255.255.0`
3. PC1 も同様に `192.168.1.20` / `255.255.255.0` を設定する

### 3-3. 通信してみる

1. PC0 の **[Desktop] タブ → Command Prompt** を開く
2. `ping 192.168.1.20` と入力して Enter
3. `Reply from 192.168.1.20 ...` と表示されれば**成功**です 🎉
   - 最初の 1〜2 行が `Request timed out.` になるのは正常です。
     もう一度実行すると全部成功します

### 3-4. 保存する

1. `File > Save As` でファイル名 `day00_氏名.pkt` として保存する

## Step 4: 完了報告（10 分）

Backlog の課題 **`[Day00] 環境構築`** で以下を行ってください。

1. ping が成功した画面のスクリーンショットをコメントに貼る
2. 保存した `day00_氏名.pkt` を課題に添付する
3. 課題の状態を「処理済み」に変更する

## うまくいかないとき

| 症状 | 対処 |
|---|---|
| ping が全部 `Request timed out.` | 両方の PC の IP アドレスとサブネットマスクを見直す。ケーブルの丸印が緑か確認 |
| ケーブルの丸印が赤いまま | ケーブル種類が Copper Straight-Through か、ポートの選び間違いがないか確認 |
| そもそもインストールできない | Step 2 のつまずきポイントを確認 → 解決しなければ課題コメントで質問 |

より詳しい操作方法・トラブル対応は、同じフォルダの
**「PacketTracer導入マニュアル」** にまとまっています。研修中も随時参照してください。

**準備ができたら、Day 1「ネットワークの全体像と OSI / TCP-IP モデル」でお会いしましょう。**
