# Week0 P5 実習手順書: 自分の PC を調べる・Packet Tracer でつないでみる

> 配置先: ドキュメント `02_実習手順書 > Week0 > P5`
> 所要時間の目安: 3 時間 ／ 使用ツール: コマンド プロンプト、Cisco Packet
> Tracer

## ゴール

- 自分の PC のコマンド プロンプトを開き、`ipconfig` で自分のネットワーク設
  定(IPv4 アドレス・サブネット マスク・デフォルト ゲートウェイ)を読み取れ
  る
- `ping` を使って、デフォルト ゲートウェイまで通信が届いていることを確認で
  きる
- Packet Tracer で PC 2 台とスイッチを配置・接続し、IP アドレスを設定して、
  ping 通信を成功させられる

講義で「住所・町内の範囲・町の出口」とたとえたネットワーク設定を、きょうは
実際の画面と実際の操作で確認します。たとえ話が本当だったことを、自分の目で
見て確かめるのが今日の実習です。

---

## 前半: 自分の PC のネットワーク設定を確認する(60 分)

### 手順 1: コマンド プロンプトを開く(10 分)

1. 画面下の検索窓に「cmd」と入力する
2. 検索結果の「コマンド プロンプト」を Enter で開く
3. 黒い画面が表示されることを確認する

### 手順 2: ipconfig を実行し、3 行を書き写す(25 分)

1. 黒い画面に `ipconfig` と打ち、Enter を押す
2. ずらっと表示された文字の中から、次の 3 行を探す
   - `IPv4 アドレス`
   - `サブネット マスク`
   - `デフォルト ゲートウェイ`
3. 見つけた 3 行の数字を、ワークシートに書き写す
   - 表示される数字は人によって違います。それが正常です
   - 同じ 3 行が Wi-Fi 用・有線用など複数回表示された場合は、いま実際につ
     ながっている接続(デフォルト ゲートウェイに数字が入っている方)を書
     き写してください。「メディアは接続されていません」の表示や、
     VirtualBox/VMware などの仮想アダプタは無視してかまいません

**📷ここでスクショ**: `IPv4 アドレス`・`サブネット マスク`・`デフォルト ゲ
ートウェイ` の 3 行が見えている状態の画面を撮影してください。

### 手順 3: デフォルト ゲートウェイへ ping を打つ(25 分)

1. 手順 2 で書き写したデフォルト ゲートウェイの IP アドレスを確認する
2. 黒い画面に `ping (デフォルトゲートウェイのIPアドレス)` と打ち、Enter を
   押す(例: `ping 192.168.1.1`)
3. `Reply from ...` が表示されることを確認する
   - これは、講義でいう「町の出口まで手紙が届いた」ことを意味します
4. `Request timed out.` しか出ない場合は、デフォルト ゲートウェイの数字を
   手順 2 で書き写した内容ともう一度見比べる
5. 数字を見比べても正しいのに `Request timed out.` のままの場合は、その
   ゲートウェイが ping に応答しない設定になっている可能性があります。あな
   たの操作ミスではないので、次の「提出物」の案内に従ってそのまま提出して
   ください

**📷ここでスクショ**: `Reply from ...` が表示されているコマンド プロンプト
の画面を撮影してください。

## 後半: Packet Tracer で PC 2 台をつないで通信させる(120 分)

### 手順 4: 事前準備を確認する(5 分)

1. 「00_ガイダンス ＞ Day00 環境構築」のページを開く
2. Packet Tracer が未インストールの人は、Day00 の Step1〜2(NetAcad アカウ
   ント作成 → インストール)をここで実施する
3. インストール済みの人は、そのまま次の手順へ進む

### 手順 5: Packet Tracer を起動する(5 分)

1. Packet Tracer を起動し、Day00 で作成したアカウントでログインする
2. 新規ファイル(空のワークスペース)を開く

### 手順 6: PC を 2 台配置する(10 分)

1. 画面左下の部品箱から `[End Devices]` → `PC` を選ぶ
2. 中央の白いワークスペースへ、2 回ドラッグして PC を 2 台置く(PC0・PC1)

### 手順 7: スイッチを 1 台配置する(5 分)

1. 同じ部品箱の `[Network Devices]` → `[Switches]` → `2960` を選ぶ
2. ワークスペースへドラッグして 1 台置く

### 手順 8: 接続アイコンとケーブルを選ぶ(5 分)

1. 画面左下の稲妻マークの接続アイコンをクリックする
2. 表示されたケーブル一覧から `Copper Straight-Through`(実線)を選ぶ

### 手順 9: PC0 とスイッチを接続する(10 分)

1. PC0 をクリックする
2. `FastEthernet0` を選ぶ
3. スイッチをクリックする
4. `FastEthernet0/1` を選び、1 本目のケーブルをつなぐ

### 手順 10: PC1 とスイッチを接続する(10 分)

1. 同じ要領で、PC1 をスイッチの `FastEthernet0/2` につなぐ

### 手順 11: ケーブルの状態を確認する(5 分)

1. ケーブル両端の丸印が緑色になるまで、30 秒ほど待つ
   - 橙色は準備中です。そのまま待てば緑になります
   - 赤色の場合は、手順 8 のケーブル種類と、手順 9〜10 のポートの選び間違い
     がないか見直す

### 手順 12: PC0 に IP アドレスを設定する(10 分)

1. PC0 をクリックする
2. `[Desktop]` タブ → `IP Configuration` を開く
3. `IP Address` に `192.168.1.10` を入力する
4. `Subnet Mask` に `255.255.255.0` を入力する

### 手順 13: PC1 に IP アドレスを設定する(10 分)

1. PC1 をクリックする
2. `[Desktop]` タブ → `IP Configuration` を開く
3. `IP Address` に `192.168.1.20` を入力する
4. `Subnet Mask` に `255.255.255.0` を入力する

### 手順 14: ping で通信を確認する(30 分)

1. PC0 をクリックする
2. `[Desktop]` タブ → `Command Prompt` を開く
3. `ping 192.168.1.20` と打ち、Enter を押す
4. `Reply from 192.168.1.20 ...` が表示されれば成功
   - 最初の 1〜2 行が `Request timed out.` になるのは正常です。もう一度実行
     すると、全部成功します

**📷ここでスクショ**: `Reply from 192.168.1.20 ...` が表示されている Packet
Tracer の Command Prompt 画面を撮影してください。

### 手順 15: 保存して提出する(15 分)

1. `File` ＞ `Save As` を選ぶ
2. ファイル名を `p5_氏名.pkt` として保存する
3. 保存した `.pkt` ファイルと、手順 14 で撮影したスクリーンショットを準備す
   る

## うまくいかないとき

| 症状 | 確認すること |
|---|---|
| `ipconfig` を打っても真っ黒な画面のまま反応がない | `ipconfig` の後ろに半角スペースの打ち間違いがないか確認する。Enter を押し忘れていないかも確認する |
| デフォルト ゲートウェイへの ping が `Request timed out.` しか出ない | 書き写した IP アドレスの数字が正しいか、手順 2 の画面ともう一度見比べる。Wi-Fi と有線で表示される値が違う場合があるため、今つながっている接続を確認する |
| IP アドレスを見比べても正しいのに、ping が `Request timed out.` のままになる | あなたの設定ミスとは限りません。ゲートウェイ側が ping に応答しない設定になっている可能性があります。その場合は timeout 画面をスクショし、「IP は確認済み」とひとことメモを添えて提出してください |
| Packet Tracer のケーブルの丸印が赤いまま緑にならない | ケーブル種類が `Copper Straight-Through` になっているか、ポートの選び間違い(`FastEthernet0` と `FastEthernet0/1` など)がないか確認する |
| Packet Tracer で ping が全部 `Request timed out.` になる | PC0・PC1 それぞれの `IP Address` と `Subnet Mask` の入力ミスがないか見直す。入力欄に余分な空白が入っていないかも確認する |
| Packet Tracer が起動しない・インストールできない | 「00_ガイダンス ＞ Day00 環境構築」のつまずきポイントを確認する |

30 分試して解決しない場合は、状況(スクリーンショット + 試したこと)を課題の
コメントに書いて質問してください。分からないことは恥ずかしいことではありま
せん。質問した人から伸びます。
