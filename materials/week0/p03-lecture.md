## 第 1 章 インターネットって、そもそも何?(全体像をつかむ)

インターネットとは、世界中のコンピュータを電線や電波でつないだ、巨大な網
(あみ)のことです。どこか 1 か所に本社や中心となる建物があるわけではありませ
ん。無数の機器が、隣の機器へ、そのまた隣の機器へと、データをバケツリレーのよ
うに渡し合うことで成り立っています。

私たちが毎日見ている「ホームページ」や「Web ページ」は、この網の上で動いてい
るサービスの一つにすぎません。LINE のメッセージも、動画配信も、メールも、同じ
網を使っています。使っているアプリは違っても、その裏側は同じ 1 つの網です。

「通信」という言葉も、ここでいったん整理しておきます。通信とは、むずかしく考
える必要はありません。データ(文字・画像などの情報)を、相手に送って、相手から
受け取ることです。これだけです。

この通信には、登場人物が 2 種類しかいません。

- 情報を出す側を **サーバ** と呼びます。お店にあたります。
- 情報を見に行く側を **クライアント** と呼びます。お客さんにあたります。

ここでもう 1 つ、**ブラウザ** という言葉も覚えておきましょう。ブラウザとは、
Web ページを見るためのアプリ(ソフト)のことです。Chrome や Edge がその代表例
です。クライアント役の機器(パソコンやスマホ)の中で、実際にページを開いて表
示してくれるのが、このブラウザです。

自分のパソコンやスマホでニュースサイトを見るとき、その機器はいつもクライアン
ト役です。ページの中身を持っているのは、どこか別の場所にあるサーバです。

今日のゴールは、「URL を打つと、その裏側で何が起きているのか」を、一本のつな
がった話として言えるようになることです。次の章から、その話を順番にたどってい
きます。

## 第 2 章 URL を打ってから画面が出るまで

### URL の中身を分解する

URL とは、見たいページの住所を書いた紙のようなものです。たとえば
`https://www.example.com/page` という URL は、次の 3 つの部分に分けられます。

| 部分 | 意味 |
|---|---|
| `https://` | 通信の約束事(データをどんな方法でやり取りするか) |
| `www.example.com` | 相手(サーバ)の名前 |
| `/page` | その中のどの部屋(ページ)か |

### 郵便を送る旅として見る

ここから先の一連の流れを、郵便を送る場面にたとえて説明します。ブラウザは郵便
局の窓口、サーバは相手の会社、DNS は会社の番号を調べる電話帳、やり取りされる
データは小分けにした手紙です。この 1 つの物語で、今日の講義の中心部分を貫きま
す。

**ステップ 1: 電話帳で宛先の番号を調べる(DNS)**

ブラウザは、URL に書かれた `www.example.com` という名前だけでは、実はまだ相手
に手紙を届けられません。名前は分かっていても、本当の宛先である番号を知らない
からです。

そこでブラウザは、まず **DNS** という電話帳に問い合わせます。「example.com さ
んの番号を教えてください」という具合です。DNS は、この問い合わせに対して番号
を教えてくれます。この番号のことを **IP アドレス** と呼びます。

IP アドレスとは、ネット上の住所(番地)のことです。`192.0.2.1` のような数字で
表され、世界中でその機器を 1 つに特定する宛先になります。人間は名前で相手を呼
びますが、機械は番号がないと配達先を特定できません。だから、名前を番号に変え
る電話帳(DNS)が必要になるのです。

**ステップ 2: 窓口が注文の手紙を送る**

宛先の番号が分かったら、ブラウザ(郵便局の窓口)は、「このページをください」と
いうお願いの手紙を、相手のサーバ(会社)あてに送ります。

**ステップ 3: 会社が返事の手紙を送り返す**

サーバは、注文どおりにページのデータ(文字・画像・見た目の指定など)を手紙に
して送り返します。

### 手紙は 1 通ではない

大きなデータは、1 通の封筒には入りきりません。そこで小分けにされ、何通にも分
かれて送られます。あなたの手元に届いたそれぞれの手紙は、パズルのように組み立
て直されて、1 つのページとして画面に表示されます。この様子は、今日の実習で実
際に目で見て確認します。

ここまでの一連の往復は、たった 1 秒足らずのあいだに自動的に行われています。だ
から、普段は意識することがありません。

## 第 3 章 家がインターネットにつながる仕組み

### ルータは家の玄関であり、仕分け係

家庭用ルータとは、家の中にある機器と、インターネットという網とをつなぐ玄関で
あり、郵便の仕分け係にあたる機器です。外から届いた手紙を、それが PC 宛てなの
かスマホ宛てなのかテレビ宛てなのかを振り分け、逆に家から出す手紙を外へ送り出
します。

1 台のルータに、PC もスマホもテレビも同時にぶら下がることができます。家の中の
機器は、みな同じ玄関を通って外に出ています。

### LAN とは

つなぎ方の話に入る前に、**LAN**(ラン)という言葉を確認しておきます。LAN と
は、ローカルエリアネットワークの略で、家庭や教室のように、限られた範囲の中だ
けをつなぐ小さな通信網のことです。この後に出てくる「LAN ケーブル」や「LAN と
いったランプ」は、どちらもこの LAN に関係する言葉です。

### つなぎ方は 2 通り

家の中の機器がルータにつながる方法には、2 通りあります。

| つなぎ方 | 方法 | 特徴 |
|---|---|---|
| 有線 | LAN ケーブルで直接つなぐ | 線でつながっていて安定・速い |
| Wi-Fi | 電波で無線でつなぐ | 線がなく便利、ときに不安定なことがある |

Wi-Fi は電波なので、壁を越えて周囲に漏れてしまいます。そのままでは、近くにい
る誰でもつながれてしまう状態になります。そこでパスワードによって鍵をかけ、決
められた人だけがつながれるようにしています。パスワードの管理など、詳しいセ
キュリティの話は P4 で扱います。

### ランプが教えてくれること

ルータの本体には、電源・インターネット・Wi-Fi・LAN といったランプが並んでいま
す。これらのランプは、今どこがつながっているかを、色や点滅で教えてくれる合図
です。通信のトラブルが起きたとき、最初に確認する場所になります。今日の実習で、
実際に自分の目でこのランプを観察します。

## 第 4 章 スマホと、外出先での通信

スマホのつなぎ方にも、2 通りあります。家では Wi-Fi につながり、外では **モバ
イル通信(4G/5G)** につながります。多くの場合、この切り替えは自動で行われてい
ます。

モバイル通信は、街なかに立っている **基地局**(電波の中継アンテナ)を経由して
います。基地局から先は、家庭用ルータの場合と同じインターネットの網につながっ
ています。つまり、スマホが外でモバイル通信を使っているときも、最終的にたどり
着く先は家のネットと同じ場所です。

「ギガが減る」という表現をよく聞きます。これは、データを送ったり受け取ったり
するたびに、決められた通信量を消費していく仕組みのことです。動画は、文字だけ
のページに比べてやり取りされるデータの量がずっと多いため、ギガの減り方も速く
なります。

カフェや駅にある無料の Wi-Fi は便利ですが、知らない人たちと同じ玄関を共有して
いる状態でもあります。そのため注意が必要です。詳しい対策は P4 で扱います。

家でもスマホでも、やっていることの中身は同じです。どちらも、データを送って、
受け取っています。

## 第 5 章 今日の言葉と、本編(Day1〜3)への橋渡し

今日出てきた言葉を、もう一度まとめて確認します。

| 言葉 | 意味 |
|---|---|
| サーバ | 情報を出す機器(お店) |
| ブラウザ | 情報を見に行く窓口(郵便局の窓口) |
| DNS | 名前を番号に変える電話帳 |
| IP アドレス | ネット上の住所を表す番号 |
| ルータ | 家とネットをつなぐ玄関・仕分け係 |

本編の Day1 では、今日「郵便の旅」として見た一連の流れを、通信の地図にあたる
OSI モデルや TCP/IP モデルという段階に分けて、より正確に学び直します。今日つ
くったイメージが、その土台になります。

Day1 で登場するルータやスイッチは、今日見た家庭用ルータの、会社向け・本格版に
あたる機器です。今度は自分でコマンドを打って設定していきます。

Day3 では、今日「住所」として眺めた IP アドレスを、自分で設計し、割り当てられ
るようになります。P2 で学んだ 2 進数の知識が、ここで生きてきます。

今日のまとめの合言葉は、次の一文です。

> URL を打つ → 電話帳で住所を調べる → 窓口が注文を送る → 会社が返事を送り返す
> → 手元で組み立てる

この一本の物語を、自分の言葉で言えるようになれば、今日の講義は合格です。

## まとめ

- インターネットは世界中の機器をつないだ 1 つの網であり、中心となる 1 か所は
  存在しない
- 情報を出す側がサーバ、見に行く側がクライアント。自分の PC やスマホはいつも
  クライアント役
- URL を打つと、DNS が名前を IP アドレス(住所)に変換し、ブラウザ(窓口)が
  サーバ(会社)にお願いの手紙を送り、サーバがデータを送り返す
- 家庭用ルータは、家の機器とインターネットをつなぐ玄関であり、仕分け係。つなぎ
  方には有線と Wi-Fi の 2 通りがある
- スマホは家では Wi-Fi、外では基地局を使うモバイル通信につながる。やっている
  ことの中身はどちらも同じ、データの送受信である
