# Week0 P1 講義: コンピュータのきほん

> 配置先: ドキュメント `01_教材 > Week0 > P1`
> 学習時間の目安: 3 時間 ／ 対象: IT リテラシーゼロからのスタート

## 学習目標

この講義を終えると、次のことができるようになります。

1. コンピュータが行っている「入力 → 処理 → 出力」という基本の流れを説明できる
2. ハードウェアの 3 つの部品(CPU・メモリ・ストレージ)の役割の違いを説明できる
3. 電源を切ったときに、メモリの中身とストレージの中身がそれぞれどうなるかを説明できる
4. OS とアプリの違いと、両者の関係を説明できる
5. ファイル・フォルダ・拡張子の意味を理解し、保存場所とファイルの種類を意識して操作できる

---

## 第 1 章 コンピュータって、そもそも何をしている機械?

最初に結論を言います。コンピュータは魔法の箱ではありません。やっていることを
突き詰めると、次の 3 つしかありません。

1. 情報を受け取る(入力)
2. 受け取った情報をもとに処理する
3. 処理した結果を外に出す(出力)

「受け取る」を担当しているのが、キーボード・マウス・マイク・カメラといった機器
です。これらは入力装置と呼ばれます。文字を打つ、画面をクリックする、声を録音
する。どれも「情報をコンピュータの中に入れる」動作です。

「出す」を担当しているのが、画面・スピーカー・プリンタといった機器です。これら
は出力装置と呼ばれます。文字や写真を表示する、音を鳴らす、紙に印刷する。どれも
「コンピュータの中の情報を外に出す」動作です。

残るは真ん中の「処理する」です。ここが今日の講義の主役になります。処理を実際に
行っているのは、コンピュータ内部にある機械の部品です。これをハードウェアと呼び
ます。

ここで、今日いちばん大きな仕分けを一度だけ示しておきます。

- **ハードウェア**: 目に見える、機械としての部品(内部の部品も、キーボードや
  画面のような周辺機器も含みます)
- **ソフトウェア**: 目に見えない、コンピュータへの指示書

この 2 つの違いは第 3 章でもう一度使います。ここでは名前だけ覚えておいてください。

今日の講義が終わるころには、自分のパソコンの中で何が起きているかが、ぼんやりと
でも見えるようになります。それが今日のゴールです。

## 第 2 章 ハードウェアの 3 要素 - 作業机でつかむ

コンピュータの内部にあるハードウェアはたくさんの部品からできていますが、今日は
その中でも特に重要な 3 つに絞ります。CPU・メモリ・ストレージです。

この 3 つは、1 つの作業机の風景にたとえると、関係がつかみやすくなります。

> **たとえるなら**: 作業する人が CPU、作業机の広さがメモリ、本棚がストレージ
> です。人は本棚から資料を取り出して机の上に広げ、作業します。机が広いほど、
> 一度にたくさんの資料を広げられます。作業が終わって席を立つと、机の上は
> 片付けられます。しかし本棚の中身はそのまま残ります。

### CPU - 実際に作業する人

CPU(Central Processing Unit、中央処理装置)は、計算や判断を実際に行う部品です。
作業机のたとえでいう「作業する人」にあたります。頭の回転が速い人ほど仕事が速い
のと同じで、CPU の性能が高いほど処理が速くなります。

CPU のカタログには「◯ GHz」という数字が書かれています。今日の時点では、この
数字は「速さのおおまかな目安」くらいに考えておけば十分です。詳しい意味は今後の
実習で扱います。

### メモリ(RAM) - 広げておく机

メモリは、いま作業中のデータを広げておく場所です。作業机のたとえでいう「机の
広さ」にあたります。机が広いほど、同時にたくさんの資料を広げられます。コン
ピュータで言えば、メモリが大きいほど、同時にたくさんのアプリを開いても動作が
重くなりにくいということです。

メモリには大事な性質があります。**電源を切ると、机の上と同じように中身が消え
ます**。

### ストレージ(SSD/HDD) - しまっておく本棚

ストレージは、写真・書類・アプリなどのデータを保管しておく場所です。作業机の
たとえでいう「本棚」にあたります。SSD や HDD と呼ばれる部品がこれにあたります。

ストレージにも大事な性質があります。**電源を切っても、本棚の中身と同じように、
データは残ります**。

### 消える机と残る本棚

ここが、初心者がいちばん混同しやすいポイントです。並べて確認します。

| 部品 | たとえ | 電源を切ると |
|---|---|---|
| メモリ(RAM) | 広げておく机 | 中身は消える |
| ストレージ(SSD/HDD) | しまっておく本棚 | 中身は残る |

「保存」という操作が必要なのは、机の上(メモリ)に広げたままの作業内容を、本棚
(ストレージ)にしまい直すためです。保存せずに電源を切ると、机の上にあったもの
はすべて消えてしまいます。

### 実物で確認する

自分のパソコンで、実際にメモリとストレージの容量を見てみます。

1. [設定]を開く(Windows の場合)
2. [システム] → [バージョン情報]でメモリの容量を、[ストレージ]でストレージの
   容量を確認する
3. それぞれ「◯ GB」という数字が表示されていることを確認する

表示されている GB という単位の詳しい意味は、次の P2「数のきほん」で扱います。
今日は「自分のパソコンにも、机の広さと本棚の大きさに相当する数字がある」ことだ
け確認できれば十分です。

## 第 3 章 OS とアプリ - 土台と、その上の道具

### OS - 電源を入れて最初に動き出す土台

パソコンの電源を入れると、いちばん最初に動き出すソフトウェアがあります。これを
OS(オペレーティングシステム)と呼びます。Windows や macOS が OS にあたります。
OS は、人間とハードウェアの間に立って、両者をつなぐ通訳であり管理役です。

### アプリ - OS の上で使う道具

アプリ(アプリケーション)は、OS の上で動く、特定の目的のための道具です。ブラウ
ザ・メール・Word・Excel などがアプリにあたります。アプリは、OS がなければ 1 つ
も動くことができません。

> **たとえるなら**: OS は電気も水道も通った家そのもの、アプリはその家に置いて
> 使う家電や家具です。家(OS)がなければ、家電(アプリ)を持ってきても使うことが
> できません。

### 実物で確認する

いま目の前にある画面そのものが、OS の画面です。並んでいるアイコンの一つひとつ
が、アプリです。実際に自分のデスクトップ画面を見ながら、次を確認してください。

- 画面全体の土台になっている部分(デスクトップ、タスクバーなど) → OS の画面
- 並んでいるアイコン(ブラウザ、メールなど) → それぞれがアプリ

### ハードウェアとソフトウェアの対応

第 1 章で示した仕分けに、ここでもう一度対応づけます。

- ハードウェア = 目に見える機械の部品(第 2 章の CPU・メモリ・ストレージなど)
- ソフトウェア = 目に見えない指示書(OS とアプリは、どちらもソフトウェアの仲間
  です)

本編の実習からは、常に「いま自分はどのアプリで作業しているか」を意識する場面が
増えます。今日はその第一歩として、OS とアプリの違いを押さえておいてください。

## 第 4 章 ファイルとフォルダ - データの置き場所を住所で考える

### ファイル - データのひとまとまり

ファイルとは、データのひとまとまりのことです。1 枚の書類、1 枚の写真、1 つの
動画。それぞれが 1 つのファイルです。ファイルには必ず名前がついています。

### フォルダ - ファイルをまとめる入れ物

フォルダとは、ファイルを入れてまとめておく入れ物です。フォルダの中に、さらに
別のフォルダを作ることもできます。これを入れ子と呼びます。

> **たとえるなら**: ファイルは 1 枚の書類、フォルダは書類を仕分けて入れる引き
> 出しです。引き出しの中に、さらに小さな仕切り箱を入れることもできます。これが
> フォルダの入れ子にあたります。

### 保存場所を住所として辿る

ファイルがどこに保存されているかは、住所のように上から辿ることができます。
たとえば「デスクトップの中の、練習フォルダの中の、memo ファイル」というように
です。代表的な保存場所には、次のようなものがあります。

- デスクトップ
- ドキュメント

### 実物で確認する

エクスプローラー(Windows)を開き、フォルダをダブルクリックして、下の階層へ下り
ていく様子を確認します。1 つ下の階層に進むごとに、住所が 1 段階詳しくなって
いきます。

ファイルが行方不明になる原因のほとんどは、住所を覚えていないことです。保存する
ときに「どこに入れたか」を見る習慣を、今のうちからつけておいてください。

## 第 5 章 拡張子 - ファイルの種類を表す名札

### 拡張子とは

ファイル名の最後についている「.◯◯」の部分を、拡張子と呼びます。拡張子は、
そのファイルがどんな種類のファイルかを表す名札です。

代表的な拡張子は次のとおりです。

| 拡張子 | ファイルの種類 |
|---|---|
| `.txt` | 文書(テキスト) |
| `.jpg` `.png` | 画像 |
| `.pdf` | PDF 文書 |
| `.docx` | Word の文書 |
| `.xlsx` | Excel の表 |

### 拡張子とアプリの関係

拡張子が違うと、そのファイルを開くときに起動するアプリも変わります。たとえば
`.docx` をダブルクリックすると Word が開き、`.xlsx` をダブルクリックすると
Excel が開きます。

このため、拡張子を勝手に書き換えると、ファイルが正しく開けなくなることがあり
ます。ファイル名を変更するときは、拡張子の部分を変えないよう注意してください。

### 実物で確認する

Windows の初期設定では、拡張子は画面に表示されません。設定を切り替えて、隠れて
いた拡張子を表示させます。

1. エクスプローラーを開く
2. 上部の[表示]タブを開く
3. [表示] → [ファイル名拡張子]にチェックを入れる
4. これまで見えていなかった `.txt` などの拡張子が、ファイル名の末尾に表示され
   るようになったことを確認する

今日はここまでで十分です。「拡張子はファイルの種類を表す名札である」という点
だけ、確実に持ち帰ってください。本編以降、ファイルや設定を扱う場面で何度も出て
きます。

## まとめ

- コンピュータは「入力 → 処理 → 出力」の 3 つを行っている機械である
- ハードウェアの 3 要素は CPU(作業する人)・メモリ(広げる机、電源を切ると消え
  る)・ストレージ(しまう本棚、電源を切っても残る)
- OS は土台となるソフト、アプリは OS の上で動く道具。どちらも目に見えない指示書
  = ソフトウェアである
- ファイルはデータのひとまとまり、フォルダはファイルをまとめる入れ物。保存場所
  は住所のように上から辿れる
- 拡張子はファイルの種類を表す名札であり、開くアプリを決めている
