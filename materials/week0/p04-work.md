# Week0 P4 実習：仕事のIT基礎（Backlog・Markdown・セキュリティ）

> 配置先: ドキュメント `02_実習手順書 > Week0 > P4`
> 所要時間の目安: 3 時間 ／ 使用ツール: Backlog、スクリーンショット

この実習では、講義で学んだBacklogの操作・Markdownの書き方・スクリーンショットの
撮り方と貼り方を、練習用の課題を使って実際に手を動かして確認します。
今日操作した手順は、明日からの提出のたびに毎回使うことになります。

## ゴール

- 練習用の課題を開き、状態を「未対応→処理中→処理済み」の順に自分で変更できる。
- コメント欄にMarkdown（見出し・箇条書き・コードブロック）を使って書き込み、プレビューで確認できる。
- スクリーンショットを撮り、コメントへの貼り付けと、添付エリアへのファイル添付の両方を行える。
- 「状況・試したこと・スクショ」の質問テンプレートを使って、実際にコメントを1件書ける。

## 準備するもの

- Backlogのログイン情報（ユーザー名・パスワード）
- 自分のPC（WindowsまたはMac）
- 練習用のサンプル画像ファイル1枚（講師が配布したもの、または自分で撮ったスクリーンショットでも可）

---

## 手順

### ステップ1：練習用課題を開く

1. Backlogにログインします。
2. 課題一覧を開き、「担当者」を自分の名前に絞り込みます。
3. 一覧から「[Day00] Backlog操作の練習」を1回クリックして開きます。

### ステップ2：状態を「処理中」に変更する

1. 課題の上部にある状態ボタン（現在は「未対応」と表示されています）をクリックします。
2. 開いたメニューから「処理中」を選びます。
3. 状態の表示が「処理中」に変わったことを目で確認します。これが着手の合図です。

📷ここでスクショ（状態が「処理中」に変わった画面）

### ステップ3：コメント入力欄を開く

1. 画面を下にスクロールし、コメント入力欄をクリックします。
2. 「入力」タブが選ばれていることを確認します。

### ステップ4：見出しを書く

1. コメント入力欄の1行目に、次の通り入力します。半角スペースを忘れないでください。

```
## きょうの練習
```

### ステップ5：箇条書きを書く

1. 見出しの行から改行し、次の3行を入力します。各行の先頭はハイフンと半角スペースです。

```
- 状態を変えた
- コメントを書いた
- ファイルを添付した
```

### ステップ6：コードブロックを書く

1. さらに改行し、バッククォート3つだけの行を入力します。
2. 次の行に、エラー文の例を1行入力します。

```
Error: Destination host unreachable
```

3. 最後にもう一度バッククォート3つだけの行を入力し、コードブロックを閉じます。

### ステップ7：プレビューで見た目を確認する

1. コメント欄の「プレビュー」タブをクリックします。
2. 見出し・箇条書き・枠付き文字の3つが、それぞれ正しく表示されているか確認します。
3. 表示が崩れている場合は、「入力」タブに戻り、半角スペースの付け忘れがないか確認して直します。

📷ここでスクショ（プレビュータブで見出し・箇条書き・コードブロックが表示された画面）

### ステップ8：スクリーンショットを撮る

1. `Windowsキー + Shift + S`（Macは `command + shift + 4`）を押します。
2. Backlogの画面の一部を、マウスでドラッグして四角く囲みます。

### ステップ9：撮った画像をコメントに貼り付ける

1. コメント欄の中をクリックします。
2. `Ctrl + V` を押し、撮ったスクリーンショットをコメントに貼り付けます。
3. プレビュータブで、画像が正しく表示されているか確認します。

**Macの場合**：ステップ8で `command + shift + 4` を使った場合、画像はクリップボードではなく
デスクトップにファイルとして保存されているため、`command + V` を押しても貼り付けられません。
その画像ファイルを、コメント欄または添付エリアにマウスでドラッグ＆ドロップしてください
（貼り付け操作は不要です）。どうしてもコメントに直接貼り付けたい場合は、ステップ8を
`command + control + shift + 4` でやり直し、画像をクリップボードにコピーしてから
`command + V` を押してください。

### ステップ10：コメントを投稿する

1. 「更新」ボタンを押して、ここまでの内容を投稿します。
2. 投稿したコメントが、画面下に表示されたことを確認します。

### ステップ11：ファイルを添付する

1. 手元の練習用画像ファイルを、課題の添付エリアにマウスでドラッグ＆ドロップします。
2. うまくいかない場合は、「ファイルを選択」ボタンから同じファイルを選びます。
3. 添付エリアにファイル名が表示されたことを確認します。

📷ここでスクショ（添付エリアにファイルが追加された画面）

### ステップ12：質問テンプレートの見出しを書く

1. 新しいコメント入力欄を開きます。
2. 次の3行を、それぞれ改行して入力します。

```
## 状況
## 試したこと
## スクショ
```

### ステップ13：質問テンプレートの中身を書いて投稿する

1. 「## 状況」の下に、何をしようとして今どうなっているかを1〜2行で書きます。
2. 「## 試したこと」の下に、調べた言葉ややってみた操作を1〜2行で書きます。
3. 「## スクショ」の下に、そのときの画面写真を貼り付けます（ステップ9と同じ操作。Macの場合は
   ステップ9のMac向け補足を参照してください）。
4. プレビューで見た目を確認してから、「更新」ボタンを押して投稿します。

📷ここでスクショ（質問テンプレートの3見出しとスクショが投稿された状態）

### ステップ14：状態を「処理済み」に変更して締める

1. 状態ボタンをクリックし、「処理中」から「処理済み」に変更します。
2. 状態の表示が「処理済み」に変わったことを確認し、課題を閉じます。

## 提出物

  1. 見出し・箇条書き・コードブロック・スクリーンショットを含む「きょうの練習」コメント

- 課題の状態が「処理済み」になっていること

## うまくいかないとき

- **状態ボタンをクリックしても選択肢が出てこない**：ボタンの位置がずれていることがあります。ページを一度更新（再読み込み）してから、もう一度クリックしてください。
- **プレビューで見出しや箇条書きが反映されない**：`# ` や `- ` の後ろの半角スペースが抜けていないか確認します。全角スペースになっていないかも合わせて確認してください。
- **スクリーンショットを撮ってもコメントに貼り付けられない**：コメント入力欄の中を1回クリックしてカーソルを入れてから、`Ctrl + V`を押してください。クリックせずに貼り付けようとすると反応しません。Macで`command + shift + 4`を使った場合はファイル保存のみでクリップボードにコピーされないため、貼り付けではなくファイルのドラッグ＆ドロップを使ってください（ステップ9のMac向け補足を参照）。
- **添付エリアにドラッグ＆ドロップしてもファイルが追加されない**：ドラッグ中にマウスのボタンを離す位置が、添付エリアの外側になっていないか確認します。うまくいかない場合は「ファイルを選択」ボタンから選ぶ方法に切り替えてください。
- **コメントを投稿したのに画面に表示されない**：「更新」ボタンを押し忘れている可能性があります。入力欄に文字が残ったままなら、まだ投稿されていません。
- **時間内に終わらない**：ステップ1〜10（状態変更・Markdown・コメント投稿）までを優先して確実に仕上げてください。ステップ11以降は宿題として持ち帰っても構いませんが、次の授業までに終わらせてから小テストに臨んでください。
