## 設問

制限時間の目安は15分です。
講義資料や実習の画面は見ずに、まず自分の力で解いてください。

**問1. Backlogの「課題」が指しているものとして、最も正しいものはどれですか。**

a. プロジェクト全体をまとめた箱のこと
b. 1つの作業や宿題のこと
c. Backlogにログインしているユーザー全員のこと
d. 課題一覧の画面そのもののこと

**問2. 課題の状態が進む正しい順番はどれですか。**

a. 処理中 → 未対応 → 処理済み → 完了
b. 未対応 → 処理済み → 処理中 → 完了
c. 未対応 → 処理中 → 処理済み → 完了
d. 未対応 → 処理中 → 完了 → 処理済み

**問3. 受講者が自分で状態を変えてよいのはどの場面ですか。**

a. 「未対応→処理中」と「処理中→処理済み」の2回
b. 「未対応→処理中」から「完了」まですべて
c. 「処理済み→完了」のみ
d. 状態は講師しか変えられない

**問4. 小テストの解答やラボの結果を提出する場所として、正しいものはどれですか。**

a. 課題のタイトル欄
b. 課題のコメント欄
c. プロジェクトの名前欄
d. 担当者の絞り込み欄

**問5. 課題にファイル（画像や`.pkt`ファイルなど）を残すための機能と、その基本操作の組み合わせとして、正しいものはどれですか。**

a. 「状態」機能を使い、状態ボタンにファイルをドラッグ＆ドロップする
b. 「添付」機能を使い、添付エリアにファイルをドラッグ＆ドロップする
c. 「コメント」機能でしかファイルは残せず、添付という機能は存在しない

**問6. Markdownで見出しを作るとき、行頭に付ける正しい書き方はどれですか。**

a. `#見出し`
b. `# 見出し`

**問7. Markdownで箇条書きを作るとき、行頭に付ける正しい書き方はどれですか。**

a. `# 項目`
b. `* * 項目`

**問8. バッククォート3つ（```）で挟んだコードブロックの用途として、最も正しいものはどれですか。**

c. 表を作るための書き方
d. 画像を貼るための書き方

**問9. スクリーンショットを撮るキーボード操作の組み合わせとして、正しいものはどれですか。**

a. Windowsは `Windowsキー+Shift+S`、Macは `command+shift+4`
b. Windowsは `command+shift+4`、Macは `Windowsキー+Shift+S`
c. Windows・Macともに `Ctrl+P`
d. Windows・Macともに `Alt+F4`

**問10.（記述式）ある状況を想定します。TMDB（映画情報のサービス）の画面を開いたが、画面が真っ白で映画が何も表示されません。この状況について、質問テンプレートの3要素「状況・試したこと・スクショ（何を撮るか）」を使って、講師に送る質問文を自分の言葉で書いてください。**

## 解答・解説（講師用・翌朝公開）

| 問 | 解答 | 解説 |
|---|---|---|
| 問1 | b | Backlogでは、1つの作業や宿題のことを「課題」と呼びます。プロジェクトは研修全体の箱、課題一覧はその課題が並んだ表です。 |
| 問2 | c | 状態は未対応→処理中→処理済み→完了の順に進みます。作業前・作業中・提出済み・確認済みという流れに対応しています。 |
| 問3 | a | 受講者が自分で変えるのは「未対応→処理中」（着手時）と「処理中→処理済み」（提出時）の2回だけです。「完了」は講師が確認してから付けます。 |
| 問4 | b | 小テストの解答やラボの結果は、課題のコメント欄に書いて提出します。タイトルや担当者欄は提出物を書く場所ではありません。 |
| 問5 | b | ファイルを課題に残す機能は「添付」です。添付エリアにファイルをドラッグ＆ドロップするか、「ファイルを選択」から選んで登録します。 |
| 問6 | b | 見出しは行頭に `# `（シャープと半角スペース）を付けて作ります。半角スペースがないと見出しとして効きません。 |
| 問7 | c | 箇条書きは行頭に `- `（ハイフンと半角スペース）を付けて作ります。 |
| 問8 | b | バッククォート3つで挟んだコードブロックは、エラーメッセージやコマンド、設定内容をそのままの形で貼るための書き方です。 |
| 問9 | a | Windowsは `Windowsキー+Shift+S`、Macは `command+shift+4` でスクリーンショットを撮影します。 |
| 問10 | （記述式・下記基準で採点） | 状況・試したこと・スクショの3要素が、それぞれ具体的な言葉で書かれていれば正解とします。 |

### 問10 採点基準（記述式）

以下の3要素が、それぞれ書かれていれば正解とします。

- **状況**：TMDBの画面を開いたが映画が表示されない、画面が真っ白になっている、といった具体的な現象が書かれている。
- **試したこと**：再読み込みしてみた、検索窓にキーワードを入れて調べてみた、など、自分で何かを調べる・試みる行動が書かれている（何も試していない場合でも、「これから何を調べるか」に触れていれば可）。
- **スクショ**：真っ白な画面のスクリーンショットを撮って添付する（または貼る）ことに触れている。

言い回しは自由です。3要素の趣旨がそれぞれ伝われば、表現の違いは減点しません。
