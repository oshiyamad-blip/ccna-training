## 第1章 なぜコンピュータは0と1しか使えないのか

コンピュータの中身は、無数の小さな「スイッチ」の集まりでできています。
まずはこの事実から話を始めます。

スイッチには「入っている（ON）」か「切れている（OFF）」かの、2つの状態しかありません。
部屋の照明スイッチと同じです。
半分だけ点いている、という状態はありません。

電気の世界では、ONとOFFのような「はっきり違う2つの状態」は正確に区別できます。
しかし、その中間（少しだけ電気が流れている、など）を正しく見分けるのは難しくなります。
そのためコンピュータは、確実に区別できる2つの状態だけを使うように作られています。

そこで人間が、ONを1、OFFを0と呼ぶことに決めました。
コンピュータが0と1しか使わないのは、難しい理由があるからではありません。
中身のスイッチが2状態しか持たない、というだけのことです。

この「0か1が1個分」のことを、これから「ビット（bit）」と呼びます。
名前だけ、ここで先に覚えておいてください。
詳しい意味は第5章で扱います。

**実物を見る**

PCの電源ランプや、キーボードのCapsLockランプを実際に点けたり消したりしてみます。
「点いている＝1」「消えている＝0」を、目で確かめてください。
この2状態が、コンピュータの全ての土台になります。

## 第2章 いつも使っている10進数を、あらためて分解する

新しい2進数の話に入る前に、毎日使っている数の数え方（10進数）の仕組みを、
言葉で説明できるようにしておきます。
ここが分かると、2進数の理解はとても早くなります。

私たちは、0から9までの10種類の数字だけで、どんなに大きな数も書き表しています。
9の次まで数えると、1桁では数字が足りなくなります。
そこで位を1つ増やして「10」と書きます。
これを「繰り上がり」と呼びます。

たとえば「253」という数を分解してみます。

```
253 = 200 + 50 + 3
    = 2×100 + 5×10 + 3×1
```

右の桁から順に「1の位」「10の位」「100の位」と呼びます。
位が1つ左へ進むごとに、その位が表す重み（大きさ）が10倍ずつ増えています。

つまり10進数とは、「10ずつまとまったら位が1つ上がる数え方」です。
どんな数も、「位の重み × その桁の数字」の足し算でできています。
この見方が、次の2進数を理解するための鍵になります。

**実物を見る**

紙に大きく「253」と書きます。
各桁を丸で囲みながら、「100が2個、10が5個、1が3個」と声に出して、自分で分解してみましょう。

## 第3章 2進数 — 0と1だけで数を書く（本日の最重要）

2進数は、使える数字が0と1の2種類だけの数え方です。
第2章で見た10進数は「10集まると位が上がる」数え方でした。
2進数は「2集まると位が上がる」数え方になります。

位の重みは、10進数が10倍ずつ増えたのと同じ理屈で、2進数では2倍ずつ増えます。
右の桁から順に並べると、次のようになります。

```
1, 2, 4, 8, 16, 32, 64, 128
```

この8個の重み「128 64 32 16 8 4 2 1」は、必ず紙に書いて手元に置いてください。
今日だけでなく、この研修全体を通じて最も多く使う道具になります。

### 2進数から10進数への変換

各桁の真下にこの重みを書きます。
桁が「1」になっているところの重みだけを、すべて足します。
桁が「0」のところは、足さずにそのまま無視します。

例：`1 1 0 0 0 0 0 1` を10進数にします。

```
桁 :   1   1   0   0   0   0   0   1
重み: 128  64  32  16   8   4   2   1
```

1になっている桁は、128と64と1です。
これらを足すと、128 + 64 + 1 = 193 になります。
したがって `11000001` は10進数の193です。

### 10進数から2進数への変換（分銅の考え方）

これは、天秤と分銅（重り）を使うイメージで考えると分かりやすくなります。
128・64・32…1という重りを、大きいものから順に「これを乗せても目標の数を超えないか」と試していきます。
乗せられれば（超えなければ）その桁を1にして、乗せた分を目標の数から引きます。
乗せられなければ（超えてしまうなら）その桁を0にして、次の重りに進みます。
これを繰り返すと、目標の数ぴったりを作ることができます。

例：10進数の100を2進数にします。

```
128 → 超えるので使えない → 0
 64 → 使える（残り 100-64=36） → 1
 32 → 使える（残り 36-32=4）  → 1
 16 → 超えるので使えない → 0
  8 → 超えるので使えない → 0
  4 → 使える（残り 4-4=0）   → 1
  2 → 使えない → 0
  1 → 使えない → 0
```

結果を並べると `01100100` になります。
したがって10進数の100は、2進数では `01100100` です。

### なぜちょうど8個の重みなのか

コンピュータは、8ビットを1つのまとまりとして扱う約束になっています。
この「8ビットのカタマリ」は、本編Day3で学ぶIPアドレスの1区切り
（たとえば `192.168.1.1` の各数字）そのものです。
今日書いた重み表は、そのままサブネット計算の道具として使うことになります。

**実物を見る**

128から1までの重みを書いたカードを机に並べます。
1になっている桁の札だけを裏返しながら、合計を声に出して計算する練習を1人で行います。

## 第4章 16進数とMACアドレス — なぜ0〜9の先にA〜Fがあるのか

2進数は正しく数を表せますが、桁数が長くなりがちです。
たとえば `11000001` のような並びは、書き写すときに1文字見間違えるだけで別の数になってしまいます。
そこで、「4桁ずつまとめて1文字で書く」方法が使われます。
これが16進数です。

16進数は、16種類の記号を使う数え方です。
0から9まではいつも通りの数字を使いますが、10個目からは数字が足りなくなります。
そこでアルファベットを借りて、次のように1文字で表します。

```
A=10, B=11, C=12, D=13, E=14, F=15
```

2進数の4桁（`0000`〜`1111`、10進数の0〜15に対応）は、ちょうど16進数の1文字（0〜F）に対応します。
そのため2進数と16進数は相性がよく、実務ではセットで使われます。

例：2進数の `11000001` を16進数にします。

```
4桁ずつ区切る: 1100  0001
1100 = 12 = C
0001 = 1
```

したがって `11000001` は16進数で `C1` と、短く書けます。

### MACアドレスの予告

PCやスマートフォンには、機器ごとに固有の「名札」が付いています。
たとえば `00-1A-2B-3C-4D-5E` のような英数字の並びで、これをMACアドレスと呼びます。
この並びが、今日学んだ16進数そのものです。
本編Day5で本格的に扱います。

**実物を見る**

上に出てきた `00-1A-2B-3C-4D-5E` をもう一度見てみましょう。この中の `1A` や `3C` が、
今日学んだ16進数そのものです。英数字が混じった見慣れない並びも、16進数だと分かれば
こわくありません。

## 第5章 bitとByte、K・M・Gの単位（本日のまとめ）

第1章で名前だけ出した「ビット（bit）」は、0か1が1個分のことでした。
このビットを8個ひとまとめにしたものを、「バイト（Byte）」と呼びます。
つまり、8ビット＝1バイトです。

卵にたとえると分かりやすくなります。
ビットは卵1個ぶんの、数える最小の粒です。
バイトは卵8個入りの、まとめて扱う1パックです。
中身はどちらも同じ卵で、粒で数えるかパックで数えるかが違うだけです。

なぜこれを覚える必要があるかというと、ファイルの大きさも、通信の速さも、
すべてこのバイトを基本の単位にして表されているからです。

大きな数は、まとめて略して呼びます。
1000倍ごとに呼び名が変わり、次のように大きくなっていきます。

```
B（バイト） → KB（キロバイト） → MB（メガバイト） → GB（ギガバイト） → TB（テラバイト）
```

おおよその対応は、次の通りです。

```
1KB ≒ 1000B
1MB ≒ 1000KB
1GB ≒ 1000MB
```

身近な例で、桁の感覚をつかんでおきます。

- 写真1枚：数MB程度
- 音楽1曲：数MB程度
- 映画1本：数GB程度
- スマートフォンの保存容量：128GBなど

最後に、注意点を1つだけ挙げます。
通信の速さは小文字の bit（bps）で表され、ファイルの大きさは大文字の Byte（B）で表されます。
同じような表記でも単位が違うと8倍の差が出ます。
今日はこの事実だけ、頭の隅に置いておいてください。

## まとめ

- コンピュータの中身は無数のスイッチの集まりで、ON/OFFの2状態だけを確実に区別できる。ONを1、OFFを0と呼ぶ。
- 10進数は「位の重み × 桁の数字」の足し算でできている。位は10倍ずつ増える。
- 2進数は同じ考え方で、位が2倍ずつ増える（128 64 32 16 8 4 2 1）。1の桁の重みを足せば10進数に直せる。
- 10進数から2進数への変換は、大きい重みから順に「引けるか」を試していく。
- 16進数は2進数4桁を1文字にまとめたもの。0〜9の次は A〜F（10〜15）を使う。
- 8ビット＝1バイト。B・KB・MB・GB・TBは1000倍ごとに呼び名が変わる。

## 確認問題

**問1. コンピュータの中身が0と1しか使わない理由として、最も正しいものはどれですか。**

a. 0と1以外の数字を人間が発明していないから
b. コンピュータの中身はスイッチの集まりで、確実に区別できるON/OFFの2状態だけを使っているから
c. 計算を速くするために0と1だけに制限しているから
d. 昔のコンピュータの名残で、今は変える予定があるから

<details><summary>解答</summary>

**b**

コンピュータの中身は無数のスイッチでできており、電気の世界で確実に区別できるのはON/OFFの2状態だけです。0と1はこの2状態に人間が付けた名前です。

</details>

**問2. 10進数の「253」を「重み × 桁の数字」の足し算で表したとき、正しいものはどれですか。**

a. 2×10 + 5×100 + 3×1
b. 2×100 + 5×10 + 3×1
c. 2×1 + 5×10 + 3×100
d. 2×100 + 5×1 + 3×10

<details><summary>解答</summary>

**b**

右の桁から「1の位」「10の位」「100の位」で、位が1つ左に進むごとに重みが10倍になります。253は2×100+5×10+3×1です。

</details>

**問3. 2進数「1100」を10進数に直すといくつですか。重み表「8 4 2 1」を使って考えてください。**

a. 10
b. 11
c. 12
d. 13

<details><summary>解答</summary>

**c**

桁の下に重みを書くと「8 4 2 1」に対して「1 1 0 0」です。1になっている桁の重みは8と4なので、8+4=12です。

</details>

**問4. 10進数の「9」を2進数に直すと、正しいものはどれですか。**

a. 1001
b. 1010
c. 1100
d. 0111

<details><summary>解答</summary>

**a**

重み「8 4 2 1」に対し、8は使える（残り1）、4は使えない、2は使えない、1は使える、で「1001」になります。

</details>

**問5. 16進数について、正しい説明はどれですか。**

a. 0〜9の10種類の記号だけを使う数え方である
b. 2進数の8桁をまとめて1文字にした数え方である
c. 2進数の4桁をまとめて1文字にした数え方で、10以上はA〜Fを使う
d. コンピュータの中では使われておらず、人間の便宜のためだけに存在する架空の数え方である

<details><summary>解答</summary>

**c**

16進数は0〜9とA〜F（10〜15）の16種類の記号を使い、2進数の4桁（0000〜1111）がちょうど16進数の1文字に対応します。

</details>

**問6. 2進数「11000001」を16進数に直すと、正しいものはどれですか。**

a. C1
b. 1C
c. B1
d. D2

<details><summary>解答</summary>

**a**

4桁ずつ区切ると「1100」と「0001」になります。1100は10進数で12、つまりC、0001は1です。並べると`C1`になります。

</details>

**問7. 「8ビット＝1バイト」という関係の説明として、正しいものはどれですか。**

a. ビットは卵8個入りのパック、バイトは卵1個の粒である
b. ビットは0か1が1個分の最小の粒で、それを8個まとめたものがバイトである
c. ビットとバイトは全く同じ意味で、呼び方が違うだけである
d. バイトは16個のビットからできている

<details><summary>解答</summary>

**b**

ビットは0か1が1個分の、数える最小の粒です。このビットを8個ひとまとめにしたものをバイトと呼びます（卵1個＝ビット、卵8個入りパック＝バイト、というたとえでした）。

</details>

**問8. データ量の単位を、小さい順に正しく並べたものはどれですか。**

a. B → KB → MB → GB → TB
b. B → MB → KB → GB → TB
c. KB → B → MB → GB → TB
d. B → KB → GB → MB → TB

<details><summary>解答</summary>

**a**

B（バイト）→ KB（キロバイト）→ MB（メガバイト）→ GB（ギガバイト）→ TB（テラバイト）の順に、1000倍ずつ大きくなっていきます。

</details>

**問9. 2進数「10101010」を10進数に直すといくつですか。重み表「128 64 32 16 8 4 2 1」を使って考えてください。**

a. 165
b. 170
c. 175
d. 180

<details><summary>解答</summary>

**b**

桁の下に重みを書くと「128 64 32 16 8 4 2 1」に対して「1 0 1 0 1 0 1 0」です。1になっている桁の重みは128・32・8・2なので、128+32+8+2=170です。

</details>

**問10. 第3章で学んだ「なぜちょうど8個の重み（128〜1）を使うのか」の理由として、最も正しいものはどれですか。**

a. 人間が覚えやすい数だから
b. コンピュータが8ビットを1つのまとまり（1バイト）として扱う約束になっているため

d. 16進数が8種類の記号を使うため

<details><summary>解答</summary>

**b**

コンピュータは8ビットを1つのまとまり（1バイト）として扱う約束になっています。この8ビットのカタマリが、本編Day3で学ぶIPアドレスの1区切りにもそのまま使われます。

</details>
