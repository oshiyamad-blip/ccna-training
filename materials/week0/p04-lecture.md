## 第1章 Backlogって何？ なぜ毎日これを使うのか（画面ツアー）

これから20日間、毎日みなさんが訪れる場所があります。
それが「Backlog」です。
Backlogは、講師と受講者が同じ画面を見ながら、今日やること・出したもの・質問と回答を
全部残しておく、仕事用のノートのような道具です。

なぜメールではなく、Backlogを使うのでしょうか。
メールは基本的に1対1でやり取りするものです。
送った内容は受け取った人の受信箱に流れていき、他の人からは見えません。
Backlogは違います。
書いた内容は、そのプロジェクトに参加している全員がいつでも見られる場所に残ります。
誰が見ても分かる形で、後からも見返せる。
だから研修の連絡・提出物・質問はすべてBacklogに集約します。

実際の画面を開いて、覚える場所は3つだけに絞ります。

1. **プロジェクト** — 「CCNA研修2026」という、研修全体を入れる箱です。まずこの箱に入ります。
2. **課題一覧** — プロジェクトの中にある、やることが並んだ表です。行が1つずつの作業に対応します。
3. **課題** — 課題一覧の行を1つクリックして開いた、1件の作業画面です。

用語を覚える必要はありません。
画面を見ながら、講師が指差す場所と、今読んだ3つの言葉が一致すれば十分です。

Backlogでは、1つの作業や宿題のことを「課題」と呼びます。
課題一覧を開くと、次のようなタイトルの行が並んでいます。

```
[Day01] 講義：ネットワークとは何か
[Day01] 小テスト：ネットワークとは何か
[Day02] ラボ：ケーブルの種類
```

`[Day01]` のような日付と、講義・小テスト・ラボといった種類が、タイトルの先頭に付いています。
これがあなたの毎日の宿題です。
一覧を見れば、今日から先、何をやることになっているか全部分かります。

自分の課題だけを見たいときは、課題一覧の「担当者」という絞り込み欄に自分の名前を選びます。
これで、自分に関係のある課題だけが表示されます。
今日から毎朝、まずこの絞り込み画面を開くところがスタート地点です。

この章のゴールは、用語を暗記することではありません。
「プロジェクト・課題一覧・課題」の3語が、画面のどこを指しているか分かれば合格です。

## 第2章 課題を1件開いてみる（状態・コメント・添付の3点セット）

課題一覧から、課題を1件開きます。
画面の上から順に、指差しで確認していきます。

- **タイトル** — その課題が何の作業かを表す一行です。
- **本文** — やることの詳しい説明です。講師が用意した指示文が書かれています。
- **状態** — 作業の進み具合を表すボタンです。次で詳しく説明します。
- **コメント欄** — 課題の下にある、みんなで書き込む会話欄です。
- **添付** — ファイルを課題にくっつけて残す場所です。

毎日みなさんが触るのは、基本的にこの5つだけです。
難しい機能ではありません。

### 状態（じょうたい）

「状態」は、その課題が今どこまで進んでいるかを示す札です。
次の4段階の順番で進みます。

```
未対応 → 処理中 → 処理済み → 完了
```

- **未対応** — まだ手をつけていない状態です。課題が新しく割り当てられると、最初はこの状態になっています。
- **処理中** — いま作業をしている状態です。
- **処理済み** — 作業を出し終わった状態です。提出が完了したことを意味します。
- **完了** — 講師が内容を確認し終えた状態です。

洗濯かごの札を思い浮かべると分かりやすくなります。
未対応は、まだ洗っていない服がかごに入っている状態です。
処理中は、いま洗濯機の中で回っている状態です。
処理済みは、干し終わって講師の点検を待っている状態です。
完了は、たたんで棚に戻された状態です。
今どこにあるか、札を見れば全員が一目で分かります。

自分で変えるのは、この4段階のうち2回だけです。

- 作業に着手したとき：「未対応」→「処理中」
- 提出し終えたとき：「処理中」→「処理済み」

「完了」は講師が確認してから付けるボタンなので、自分では触りません。
役割はここではっきり分かれています。
状態ボタンは何度でも押し直せます。
押し間違えても元に戻せるので、怖がらずに操作して構いません。

### コメント

コメントは、その課題の下にみんなで書き込む会話欄です。
小テストの解答も、ラボの結果も、質問も、提出物は全部このコメントに書きます。
本編が始まると、毎日の提出先はほぼこのコメント欄になります。

### 添付（てんぷ）

添付は、ファイル（画像やPacket Tracerの保存ファイルなど）を課題にくっつけて残す機能です。
本編Day1以降、ラボで作った完成ファイル（拡張子が`.pkt`のファイル）やスクリーンショットを、
ここに貼って提出することになります。

### コメントは「更新」で初めて届く

コメント欄に文字を入力しただけでは、まだ誰にも届いていません。
入力欄の下にある「更新」ボタンを押して、初めて投稿されます。
当たり前のようですが、押し忘れると講師には何も見えません。
必ず投稿してから、画面下にコメントが表示されたことを確認してください。

## 第3章 Markdown（マークダウン）で読みやすく書く

コメントに長い文章をそのままベタ書きすると、読みにくくなります。
文字の前後に、いくつかの記号をちょっと足すだけで、見出し・箇条書き・表・枠付きの文字に
整えることができます。
講師が読みやすくなれば、返事も早くなります。
この記号のルールのことを、Markdown（マークダウン）と呼びます。

Backlogのコメント入力欄には、「入力」と「プレビュー」の2つのタブがあります。
入力タブは記号がそのまま見える画面です。
プレビュータブは、実際にどう表示されるかを確認できる画面です。
書いたら必ずプレビューで見た目を確認する習慣をつけてください。
記号の書き方が正しいかどうか、目で確認できます。

これから3つの書き方を練習します。
覚え方は「見本を真似する」で十分です。
暗記する必要はありません。

手書きのメモで、大事な言葉に下線を引いたり、行の頭に「・」を付けて箇条書きにしたりすることが
あります。
Markdownは、それと同じことをコンピュータの文字でやっているだけです。
文字の前後に記号をちょっと足して、見た目を整えています。

### 見出し

行の先頭に `# `（シャープと半角スペース）を付けると、大きな見出しになります。

```
# 大きい見出し
## 少し小さい見出し
### もっと小さい見出し
```

シャープの数が増えるほど、見出しは小さくなります。
一番よくある失敗は、シャープの後ろの半角スペースを忘れることです。
スペースがないと、見出しとして効きません。
書いたらプレビューで、文字が大きく表示されているか確認してください。

### 箇条書き

行の先頭に `- `（ハイフンと半角スペース）を付けると、点付きの箇条書きになります。

```
- 状態を変えた
- コメントを書いた
- ファイルを添付した
```

手順のように順番を示したいときは、`1. ` のように数字とピリオドで番号付きにもできます。

```
1. ログインする
2. 課題を開く
3. 状態を変える
```

### 表

縦棒 `|` で項目を区切り、2行目に `|---|---|` を入れると表になります。

```
| 項目 | 状態 |
|---|---|
| 講義 | 処理済み |
| 小テスト | 未対応 |
```

表は最初から自分で組み立てるのは難しい書き方です。
まずは配布される見本を1つ手元に置いて、その通りに真似することから始めてください。

### コードブロック

バッククォート3つ ``` ``` で挟んだ行は、等幅フォントの枠付き文字になります。

````
```
Error: Destination host unreachable
```
````

エラーメッセージや、パソコンに打ち込んだコマンド、設定内容をそのまま貼るときに使う書き方です。
本編でネットワーク機器のコマンドやエラー画面の文字を貼るとき、毎回この書き方を使うことになります。
今日習う記号の中で、一番出番の多い書き方です。
1行だけの短い言葉を枠で囲みたいときは、バッククォート1つ `` ` `` で挟むだけでも構いません。

この章は暗記が目的ではありません。
見出し・箇条書き・コードブロックの3つの見本を手元に置き、実習でそれを見ながら打てれば十分です。

## 第4章 スクリーンショット（画面写真）を撮って貼る

「動きません」と言葉だけで伝えるより、そのときの画面をそのまま見せる方が早く伝わります。
文章で状況を説明するのに5分かかっても伝わらないことが、画面1枚で一瞬で伝わることがあります。
本編のラボでは、作業結果やエラー画面をスクリーンショットで提出する場面が毎日出てきます。

スクリーンショットとは、いま画面に映っているものをそのまま画像として保存する機能です。
カメラは要りません。
キーボードの操作だけで撮影できます。

### 撮り方（Windows）

1. キーボードの `Windowsキー + Shift + S` を同時に押します。
2. 画面全体が少し暗くなります。
3. 撮りたい範囲をマウスでドラッグして、四角く囲みます。
4. 囲んだ部分が画像としてコピーされます。

### 撮り方（Mac）

1. キーボードの `command + shift + 4` を同時に押します。
2. 十字のカーソルに変わります。
3. 撮りたい範囲をドラッグして囲みます。
4. 囲んだ部分が画像ファイルとして、デスクトップに自動で保存されます。

自分のPCがWindowsかMacか、実習の前に確認しておいてください。

### 貼り方その1（コメントに直接貼り付ける）

Windowsで範囲を撮った直後は、画像がコピーされた状態になっています。
Backlogのコメント欄をクリックしてから `Ctrl + V` を押すと、画像がそのままコメントに
貼り付きます。
プレビュータブで、画像が表示されているか確認してください。

Macの `command + shift + 4` は画像がファイル保存されるだけでコピーはされないため、
この方法では貼り付けられません。Macの場合は、次の「貼り方その2」でファイルとして
添付してください。どうしてもコメントに直接貼り付けたい場合は、`command + control +
shift + 4` で撮ると画像がクリップボードにコピーされるので、その直後に `command + V`
で貼り付けられます。

### 貼り方その2（ファイルとして添付する）

Macのように画像がファイルとして保存された場合は、そのファイルを課題の添付エリアに
マウスでドラッグ＆ドロップします。
「ファイルを選択」ボタンから選ぶ方法でも構いません。
ラボの提出ファイル（`.pkt`ファイルなど）も、この方法で添付します。

### 撮るときの注意

撮影する前に、2点だけ確認してください。

1. 画面全体ではなく、関係のある部分だけを囲む。
2. パスワードなど、見せたくない情報が写り込んでいないか、貼る前にもう一度確認する。

## 第5章 分からないときの動き方（検索のコツ＋質問テンプレート）

研修全体を通じてのルールを、先にひとつ共有します。
分からないことがあるのは、恥ずかしいことではありません。
むしろ質問した人から伸びていきます。
ただし、質問する前にまず自分で15〜30分は調べてみてください。
これを「30分ルール」と呼びます。
この章では、その「調べる→それでもダメなら質問する」という動き方を身につけます。

### 検索のコツ その1：エラー文はそのまま貼る

英語の赤い文字など、エラーメッセージが出たときは、意味を考える前に、
その文をそのままコピーして検索窓に貼ります。
同じエラーで困った人の解決策が、検索結果に出てくることがよくあります。
自分で意味を訳そうとするより、まずこの方法を試してください。

### 検索のコツ その2：単語をスペースで区切る

日本語で調べるときは、「やりたいこと＋道具の名前」を短い単語に区切って入れます。

```
Backlog コメント 画像 貼り方
```

長い文章のまま検索するより、単語をスペースで並べる方がヒットしやすくなります。

### 質問のダメな例

分からないことを講師に聞くとき、次のような質問は避けてください。

```
できません。
動きません。
```

これだけでは、講師には何が起きているのか分かりません。
結局、講師が聞き返すことになり、余計に時間がかかってしまいます。

### 良い質問の型（テンプレート）

次の3点セットを、課題のコメントに書く形を覚えてください。

1. **状況** — 何をしようとして、今どうなっているか。
2. **試したこと** — 自分で何を調べたか、何をやってみたか。
3. **スクショ** — そのときの画面写真。

この3点が揃っていれば、講師はすぐに状況を把握して答えられます。
この型は、本編Day1〜20のすべての質問で使う共通の書き方です。
今日この型で一度書いておくと、明日からは迷わず質問できます。

## 第6章 仕事のセキュリティの基礎（パスワード）

ネットワークの勉強を始める前に、まず自分のアカウントを守る話をします。
Backlogのようなサービスも、メールやクラウドサービスのような外部サービスも、入口の鍵はパスワードです。
ここが弱いと、この先どれだけ知識を積んでも土台が崩れてしまいます。

### ルール1：使い回さない

同じパスワードをあちこちのサービスで使い回すと、そのうちどれか1つが漏れた瞬間に、
すべての入口が開いてしまいます。

これは、家の玄関・車・会社のロッカー・銀行の金庫を、全部同じ鍵にするようなものです。
鍵を1本すられただけで、家も車もロッカーも金庫も、全部いっぺんに開けられてしまいます。
だから、サービスごとに違う鍵（パスワード）にします。
これが一番大事な原則です。

### ルール2：長くする

短い文字列は、機械による総当たりの試行にすぐ破られてしまいます。
数字4桁の暗証番号よりも、意味のない単語をいくつかつなげた長い文の方が、
はるかに破られにくくなります。
これをパスフレーズと呼びます。
記号を複雑にすることよりも、まず長さを確保することの方が効果があります。
これが今のセキュリティの考え方です。

### ルール3：メモの置き場所に気をつける

パスワードを全部覚えきるのは、誰にとっても大変なことです。
覚えきれないのは当たり前です。
ただし、付箋に書いてモニタに貼るのは絶対にやめてください。
誰でも見える場所にあるからです。
現実的な対策としては、パスワード管理アプリを使うか、鍵のかかる場所にまとめて保管する
という方法があります。

### やってはいけないこと

次の3つは、具体的にやってはいけないこととして覚えてください。

1. パスワードを人に教える、またはチャットに書く。
2. 「あなたのパスワードを確認します」というメールやリンクを信じる。これはフィッシング（偽の
   サイトに誘導して情報を盗む手口）の入口です。
3. 公共のパソコンでログインしたまま、その場を離れる。

### 本編への橋渡し

本編Day2では、ルータやスイッチという機器に「enable secret」というコマンドで
パスワードを設定する場面が出てきます。
機器のパスワードも、今日学んだ3つのルール（使い回さない・長くする・置き場所に気をつける）と
考え方は同じです。
今日の内容は、Week0全体の締めくくりであると同時に、本編に続く土台になります。

## まとめ

- Backlogは、講師と受講者が「今日やること・出したもの・質問と回答」を残しておく仕事用の場所。覚える場所は「プロジェクト」「課題一覧」「課題」の3つ。
- 課題の「状態」は未対応→処理中→処理済み→完了の順に進む。自分で変えるのは「未対応→処理中」「処理中→処理済み」の2回だけ。
- 提出物や質問はコメント欄に書く。書いただけでは届かず、「更新」ボタンを押して初めて投稿される。ファイルは添付エリアに残す。
- Markdownは記号を少し足すだけで読みやすくする書き方。見出し（`# `）、箇条書き（`- `）、コードブロック（```` ``` ````）の3つをまず覚える。
- スクリーンショットは、Windowsは `Windowsキー+Shift+S`、Macは `command+shift+4` で撮る。コメントへの直接貼り付け、または添付エリアへのドラッグ＆ドロップで提出する。
- 分からないときは、まず自分で15〜30分調べる。それでも分からなければ「状況・試したこと・スクショ」の3点セットで質問する。
- パスワードは「使い回さない」「長くする」「置き場所に気をつける」の3ルールを守る。

## 確認問題

**問1. Backlogの「課題」が指しているものとして、最も正しいものはどれですか。**

a. プロジェクト全体をまとめた箱のこと
b. 1つの作業や宿題のこと
c. コメント欄に書き込まれた文章1件のこと
d. Backlogにログインしているユーザーのこと

<details><summary>解答</summary>

**b**

Backlogでは、1つの作業や宿題のことを「課題」と呼びます。課題一覧には、`[Day01] 講義:…`のように、日々の作業が課題として並んでいます。

</details>

**問2. 課題の「状態」について、正しい説明はどれですか。**

a. 未対応→処理中→処理済み→完了の順に進み、「完了」も自分で押してよい
b. 未対応→処理中→処理済み→完了の順に進み、自分で変えるのは「未対応→処理中」「処理中→処理済み」の2回だけ
c. 処理中→未対応→処理済み→完了の順に進む
d. 状態は一度変えると二度と戻せない

<details><summary>解答</summary>

**b**

状態は未対応→処理中→処理済み→完了の順に進みます。「完了」は講師が確認してから付けるため、自分では触りません。押し間違えても戻せるので怖がる必要はありません。

</details>

**問3. Markdownで見出しを作るとき、行頭に付ける正しい書き方はどれですか。**

a. `#見出し`（シャープの後ろにスペースなし）
b. `# 見出し`（シャープと半角スペースの後に文字）
c. `*見出し*`
d. `見出し#`

<details><summary>解答</summary>

**b**

見出しは行頭に `# `（シャープと半角スペース）を付けて作ります。半角スペースを忘れると見出しとして効かないので注意が必要です。

</details>

**問4. Windowsでスクリーンショットを撮るキーボード操作はどれですか。**

a. `Ctrl + C`
b. `command + shift + 4`
c. `Windowsキー + Shift + S`
d. `Alt + Tab`

<details><summary>解答</summary>

**c**

Windowsでは `Windowsキー + Shift + S` を押すと画面が暗くなり、ドラッグで範囲を囲んでスクリーンショットを撮れます。`command + shift + 4` はMacの操作です。

</details>

**問5. パスワードの扱い方として、正しいものはどれですか。**

a. 覚えやすいように、すべてのサービスで同じパスワードを使う
b. サービスごとに違うパスワードにし、短い数字よりも長いパスフレーズを使う
c. 忘れないように、パスワードを付箋に書いてモニタに貼っておく
d. 「パスワードを確認します」というメールが届いたら、指示通りにすぐ入力する

<details><summary>解答</summary>

**b**

パスワードは使い回さず、サービスごとに変えます。短い数字の羅列より、意味のない単語をつなげた長いパスフレーズの方が破られにくくなります。付箋にモニタへ貼る、フィッシングメールの指示に従うのは、どちらもやってはいけないことです。

</details>
