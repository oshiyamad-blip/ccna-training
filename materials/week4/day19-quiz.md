## 設問

**Q1.** 新規リソースを作成する（CRUD の Create に対応する）HTTP メソッドはどれか。

- A. GET
- B. DELETE
- C. POST
- D. PUT

**Q2.** POST リクエストによってリソースの作成に成功したとき、一般的に返される
HTTP ステータスコードはどれか。

- A. 200 OK
- B. 201 Created
- C. 400 Bad Request
- D. 500 Internal Server Error

**Q3.** SDN における南向きインターフェース（SBI）の役割として正しいものはどれか。

- A. アプリケーションとコントローラを連携させる
- B. リソースの場所を URI で定義する
- C. コントローラが配下のネットワーク機器を制御する
- D. データを JSON 形式で表現する

**Q4.** VXLAN の説明として正しいものはどれか。

- A. 物理ケーブルや IGP による IP 到達性を提供するアンダーレイ技術である
- B. コントローラとアプリケーションを接続するインターフェースである
- C. HTTP ステータスコードの一種である
- D. L2 フレームを UDP でカプセル化し、離れた拠点間で L2 セグメントを延伸するオーバーレイ技術である

**Q5.** 次の JSON データについて、`interfaces` 配列のインデックス 1（2 番目）の
要素の `status` キーの値はどれか。

```json
{
  "hostname": "SW2",
  "interfaces": [
    { "name": "Gi0/0", "status": "up" },
    { "name": "Gi0/1", "status": "down" }
  ]
}
```

- A. down
- B. up
- C. null
- D. Gi0/1

**Q6.** Ansible の特徴として正しいものはどれか。

- A. エージェントベースで Ruby の DSL を使用し、Pull 型で動作する
- B. エージェントレスで YAML 形式の Playbook を用い、SSH 経由の Push モデルで設定を配信する
- C. HCL を用いて宣言的にクラウドインフラをプロビジョニングする
- D. 対象機器に常駐エージェントを必要とし、Pull 型で設定を取得する

**Q7.** 宣言的にインフラの状態を記述し、HCL（HashiCorp Configuration Language）を
使用する IaC（Infrastructure as Code）ツールはどれか。

- A. Ansible
- B. Puppet
- C. Terraform
- D. Chef

**Q8.** 従来型（分散型）ネットワークと比較した際の、コントローラベース
ネットワークの利点として最も適切なものはどれか。

- A. 各機器が自律的に経路計算を行うため設定変更が不要になる
- B. データプレーンの機能がすべて不要になる
- C. SBI を使わずに機器を制御できるようになる
- D. ポリシーを一元的に適用でき、迅速なプロビジョニングが可能になる

**Q9.** 予測 AI（Predictive AI）の説明として正しいものはどれか。

- A. 過去のデータのパターンから将来の障害発生やトラフィック量を予測する
- B. 新しい設定案や要約などのコンテンツを生成する
- C. 人間による検証なしに出力の正しさを保証する
- D. YAML 形式で Playbook を自動生成する専用ツールである

**Q10.**（記述）REST API と JSON を用いて、ネットワークコントローラから機器情報を
取得する一連の流れを説明せよ。使用する HTTP メソッド・データ形式・認証の考え方・
成功時に返されるステータスコードに触れること。

## 解答・解説（翌日公開・講師用）

| 問 | 解答 | 解説 |
|---|---|---|
| Q1 | C | POST は新規リソースの作成（CRUD の Create）に対応する。GET は読み取り、DELETE は削除、PUT は置き換え |
| Q2 | B | 201 Created は新規作成の成功を表す。200 OK は取得成功、400/500 はエラー |
| Q3 | C | SBI（南向き）はコントローラが配下の機器を制御するためのインターフェース。アプリとの連携は NBI（北向き） |
| Q4 | D | VXLAN は L2 フレームを UDP（L3）でカプセル化し、拠点間で L2 セグメントを延伸する代表的なオーバーレイ技術 |
| Q5 | A | `interfaces` 配列のインデックスは 0 始まり。インデックス 1 は 2 番目の要素 `{"name": "Gi0/1", "status": "down"}` なので `down` |
| Q6 | B | Ansible はエージェントレス・YAML の Playbook・SSH ベースの Push モデルが特徴。Puppet/Chef はエージェントベースの Pull 型 |
| Q7 | C | Terraform は HashiCorp 製で HCL を使用し、宣言的にインフラをプロビジョニングする IaC ツール |
| Q8 | D | コントローラベースは設定の一貫性・迅速なプロビジョニング・ポリシーの一元適用が利点。データプレーンや SBI が不要になるわけではない |
| Q9 | A | 予測 AI は過去データのパターンから将来を予測する（障害予兆検知・トラフィック予測など）。コンテンツ生成は生成 AI の役割 |
| Q10 | 例 | 「コントローラに対して GET メソッドで機器情報を要求し、事前にトークンなどで認証を行い、レスポンスは JSON 形式で返る。成功時は 200 OK が返り、JSON ボディをオブジェクト・配列として解析して機器情報を取得する」等、メソッド（GET）・データ形式（JSON）・認証・成功時のステータスコード（200 系）に触れていれば正解 |

**採点**: 1 問 10 点、70 点未満は翌朝再テスト。Q10 は趣旨が合っていれば 10 点。
