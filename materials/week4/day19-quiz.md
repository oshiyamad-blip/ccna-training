## 設問

**Q1.** SDN における南向きインターフェース（SBI）の役割として正しいものはどれか。

- A. アプリケーションとコントローラを連携させる
- B. リソースの場所を URI で定義する
- C. コントローラが配下のネットワーク機器を制御する
- D. データを JSON 形式で表現する

**Q2.** Cisco Catalyst Center で、機器やクライアント・アプリの健全性をダッシュボードで
可視化し、問題の予兆を検知するワークフローはどれか。

- A. Design
- B. Policy
- C. Provision
- D. Assurance

**Q3.** VXLAN の説明として正しいものはどれか。

- A. 物理ケーブルや IGP による IP 到達性を提供するアンダーレイ技術である
- B. STP（スパニングツリープロトコル）を用いて L2 セグメントを拠点間に延伸する技術である
- C. GRE ヘッダのみで L2 フレームをカプセル化し、UDP ポートを使用しない技術である
- D. L2 フレームを UDP でカプセル化し、離れた拠点間で L2 セグメントを延伸するオーバーレイ技術である

**Q4.** 新規リソースを作成する（CRUD の Create に対応する）HTTP メソッドはどれか。

- A. PUT（リソース全体を置き換える）

**Q6.** 認証情報が無い、または誤っているためにリクエストが拒否されたことを表す

## 解答・解説（翌日公開・講師用）

| 問 | 解答 | 解説 |
|---|---|---|
| Q1 | C | SBI（南向き）はコントローラが配下の機器を制御するためのインターフェース。アプリとの連携は NBI（北向き） |
| Q2 | D | Assurance は機器・クライアント・アプリの健全性を可視化し問題の予兆を検知するワークフロー。Design は事前設計、Policy はポリシー定義、Provision は設定展開を担う。この 4 ワークフローが、意図（intent）を具体的な設定へ変換する IBN を支える |
| Q3 | D | VXLAN は L2 フレームを UDP（宛先ポート 4789）でカプセル化し、拠点間で L2 セグメントを延伸する代表的なオーバーレイ技術（MAC-in-UDP）。A はアンダーレイの説明。B は STP（L2 ループ防止技術）との混同。C は NVGRE/GRE との混同で、VXLAN は UDP を使用する点が異なる |
| Q4 | C | POST は新規リソースの作成（CRUD の Create）に対応する。PUT はリソース全体の置き換え、PATCH は部分更新（いずれも Update）、GET は読み取り（Read） |
| Q5 | B | 201 Created は新規作成の成功を表す。200 OK は取得成功、400/500 はエラー |
| Q6 | B | 401 Unauthorized は認証エラー（あなたが誰か確認できない・未ログイン）。403 Forbidden は認可エラー（認証済みだが権限が無い）。404 は指定リソースが存在しない。400 はリクエスト自体の構文・内容の誤り。「認証（401）→認可（403）の順」で整理すると覚えやすい |
| Q7 | A | `interfaces` 配列のインデックスは 0 始まり。インデックス 1 は 2 番目の要素 `{"name": "Gi0/1", "status": "down"}` なので `down` |
| Q8 | C | A は末尾カンマがあり構文エラー。B はキー `hostname` が二重引用符で囲まれておらず不可。D は文字列にシングルクオートを使っており不可（JSON の文字列は必ず二重引用符）。C がキー・値ともに正しい構文 |
| Q9 | B | Ansible はエージェントレス・YAML の Playbook・SSH ベースの Push モデルが特徴。Puppet/Chef はエージェントベースの Pull 型 |
| Q10 | C | Terraform は HashiCorp 製で HCL を使用し、宣言的にインフラをプロビジョニングする IaC ツール |
| Q11 | D | コントローラベースは設定の一貫性・迅速なプロビジョニング・ポリシーの一元適用が利点。データプレーンや SBI が不要になるわけではない |
| Q12 | A | 予測 AI は過去データのパターンから将来を予測する（障害予兆検知・トラフィック予測など）。コンテンツ生成は生成 AI の役割 |
| Q13 | 例 | 「コントローラに対して GET メソッドで機器情報を要求し、事前にトークンなどで認証を行い、レスポンスは JSON 形式で返る。成功時は 200 OK が返り、JSON ボディをオブジェクト・配列として解析して機器情報を取得する」等、メソッド（GET）・データ形式（JSON）・認証・成功時のステータスコード（200 系）に触れていれば正解 |

**採点**: 1 問 10 点（130 点満点）、91 点未満（70% 未満）は翌朝再テスト。Q13 は趣旨が合っていれば 10 点。
