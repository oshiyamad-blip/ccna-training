# Day 16 小テスト: セキュリティの概念とデバイス保護

> ルール: 10 問 / 30 分 / 教材参照なし。解答はコメントに「Q1: A」形式で提出。

## 設問

**Q1.** 「システムに存在する弱点そのもの」を指す用語はどれか。

- A. 脅威（Threat）
- B. 脆弱性（Vulnerability）
- C. エクスプロイト（Exploit）
- D. 緩和策（Mitigation）

**Q2.** DoS 攻撃によってサーバが正規利用者からの要求に応答できなくなった。CIA トライアド
のうち、直接侵害された要素はどれか。

- A. 機密性
- B. 完全性
- C. 真正性
- D. 可用性

**Q3.** 偽の DHCP サーバを社内ネットワークに設置し、誤ったデフォルトゲートウェイ情報を
クライアントに配布する攻撃はどれか。

- A. DHCP スプーフィング
- B. ARP ポイズニング
- C. IP スプーフィング
- D. MAC スプーフィング

**Q4.** 通信経路の途中に攻撃者が割り込み、通信内容を盗聴・改ざんする攻撃（MITM）に対する
最も有効な緩和策はどれか。

- A. パスワードの定期変更
- B. 送信元ポート番号のランダム化
- C. 通信の暗号化（HTTPS/IPsec など）
- D. ファイル拡張子の制限

**Q5.** 多数のボットネット化された端末から同時に攻撃対象へトラフィックを送りつけ、
サービスを利用不能にする攻撃はどれか。

- A. DoS
- B. DDoS
- C. MITM
- D. フィッシング

**Q6.** 入退室管理された部屋に、認証済みの社員に続いて無許可の人物が一緒に入室する
ソーシャルエンジニアリングの手口を何と呼ぶか。

- A. ビッシング
- B. プリテキスティング
- C. スミッシング
- D. テールゲーティング

**Q7.** ネットワーク機器にログインしたユーザが、どのコマンドを実行できるかを制御する
AAA の機能はどれか。

- A. 認可（Authorization）
- B. 認証（Authentication）
- C. アカウンティング（Accounting）
- D. 監査（Audit）

**Q8.** 802.1X によるポートベース認証で、オーセンティケータ（スイッチ）が認証情報の
検証を委ねる認証サーバとして一般的に使われるプロトコルはどれか。

- A. TACACS+
- B. RADIUS
- C. HTTPS
- D. SNMP

**Q9.** パスワードと指紋認証を組み合わせたログイン方式が、多要素認証（MFA）として
成立する理由として最も適切なものはどれか。

- A. 認証要素の数が 2 つ以上だから
- B. 異なる種類（知識と生体）の要素を組み合わせているから
- C. 両方とも生体情報だから
- D. パスワードを 2 回入力しているから

**Q10.** VTY 回線で Telnet を拒否し、SSH のみを許可する設定はどれか。

- A. `transport input telnet`
- B. `transport input ssh`
- C. `login block-for 60 attempts 3 within 30`
- D. `service password-encryption`

## 解答・解説（翌日公開・講師用）

| 問 | 解答 | 解説 |
|---|---|---|
| Q1 | B | 脆弱性はシステムの弱点そのもの。それを突く手段がエクスプロイト、突く可能性がある危険が脅威、対策が緩和策 |
| Q2 | D | DoS はリソースを枯渇させ、正規利用者がサービスを使えなくする。これは可用性への攻撃 |
| Q3 | A | 偽の DHCP サーバでゲートウェイ情報等を配布するのは DHCP スプーフィング。ARP ポイズニングは ARP 応答の偽装 |
| Q4 | C | MITM への根本的な緩和策は通信の暗号化。暗号化されていれば盗聴されても内容を読み取られない |
| Q5 | B | 分散した多数のボット端末から同時に攻撃するのが DDoS。単一発信源からの DoS と区別する |
| Q6 | D | テールゲーティングは、認証済みの人にくっついて物理的に不正入室する手口 |
| Q7 | A | 「何をすることを許すか」を制御するのは認可。認証は本人確認、アカウンティングは記録 |
| Q8 | B | 802.1X の認証サーバには一般的に RADIUS が使われる。RADIUS は UDP を使用し、パスワード部分のみ暗号化して認証と認可を統合処理する点が、TCP 49・パケット全体暗号化・柔軟なコマンド認可が特徴の TACACS+ と異なる |
| Q9 | B | MFA は異なる種類（知識・所持・生体）の要素の組み合わせで成立する。要素の数だけでは不十分 |
| Q10 | B | `transport input ssh` を VTY に設定すると、Telnet 接続を拒否し SSH のみを許可する。C はブルートフォース対策、D はパスワードの Type 7 難読化であり、いずれも Telnet/SSH の許可設定ではない |

**採点**: 1 問 10 点、70 点未満は翌朝再テスト。
