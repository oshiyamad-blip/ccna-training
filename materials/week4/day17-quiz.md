## 設問

**Q1.** プレフィックス長 /27 のネットワークに対応するワイルドカードマスクはどれか。

- A. 0.0.0.63
- B. 0.0.0.31
- C. 0.0.0.15
- D. 0.0.0.7

**Q2.** `192.168.1.5 0.0.0.0` という ACL の送信元指定と同じ意味を表す記述はどれか。

- A. `any 192.168.1.5`
- B. `192.168.1.5 255.255.255.255`
- C. `any`
- D. `host 192.168.1.5`

**Q3.** 標準 ACL の番号範囲として正しい組み合わせはどれか。

- A. 1〜99、1300〜1999
- B. 100〜199、2000〜2699
- C. 1〜99、2000〜2699
- D. 100〜199、1300〜1999

**Q4.** ACL の評価順序に関する記述として正しいものはどれか。

- A. 一致する ACE が複数あれば、最も条件が広い行が優先される
- B. ACE は常にリストの末尾から評価される
- C. 最初に一致した ACE で処理が確定し、それ以降の行は評価されない
- D. 一致する ACE がなければ、パケットはそのまま通過する

**Q5.** ある拡張 ACL に permit 文を 1 つも含めず、deny 文しか記述しなかった場合の動作として正しいものはどれか。

- A. すべての通信が許可される
- B. 暗黙の deny any により、事実上すべての通信が拒否される
- C. ACL 自体が無効になり、インターフェースに適用されない
- D. 設定時にエラーとなり、コマンドが拒否される

**Q6.** 標準 ACL と拡張 ACL の適用位置に関する原則として正しいものはどれか。

- A. 標準 ACL は送信元の近く、拡張 ACL は宛先の近くに適用する
- B. 両方とも宛先の近くに適用する
- C. 両方とも送信元の近くに適用する
- D. 標準 ACL は宛先の近く、拡張 ACL は送信元の近くに適用する

**Q7.** 拡張 ACL の構文として正しい記述順序はどれか。

- A. `permit tcp <送信元> <宛先> eq <ポート>`
- B. `permit tcp <宛先> <送信元> eq <ポート>`
- C. `permit tcp eq <ポート> <送信元> <宛先>`
- D. `permit tcp <送信元> eq <ポート> <宛先>`

**Q8.** HTTPS で使用される TCP ポート番号はどれか。

- A. 80
- B. 22
- C. 443
- D. 23

**Q9.** VTY 回線へのアクセスを特定のホストのみに制限するために、正しい設定モードとコマンドの組み合わせはどれか。

- A. インターフェースコンフィグモードで `ip access-group 10 in`
- B. line vty コンフィグモードで `access-class 10 in`
- C. グローバルコンフィグモードで `ip access-list vty 10 in`
- D. line vty コンフィグモードで `access-group vty 10 in`

**Q10.**（記述）VLAN20（192.168.20.0/24）から社内サーバ 192.168.30.200 への
HTTP（TCP/80）のみを許可し、それ以外の通信はすべて遮断したい。この要件を満たす
拡張名前付き ACL を、必要な permit / deny 行・ワイルドカードマスク・適用する
インターフェースと方向まで含めて記述し、その配置理由（送信元近く／宛先近くの
どちらに置くか）を説明せよ。

## 解答・解説（翌日公開・講師用）

| 問 | 解答 | 解説 |
|---|---|---|
| Q1 | B | /27 のサブネットマスクは 255.255.255.224。255.255.255.255 から引くと 0.0.0.31 |
| Q2 | D | ワイルドカードマスク 0.0.0.0（全ビット一致）はホスト 1 台の指定で、`host` キーワードと等価 |
| Q3 | A | 標準 ACL は 1〜99（基本範囲）と 1300〜1999（拡張範囲）。100〜199・2000〜2699 は拡張 ACL |
| Q4 | C | ACL は上から順に評価され、最初に一致した ACE で処理が確定し以降の行は評価されない（first match） |
| Q5 | B | すべての ACL の末尾には暗黙の deny any が存在するため、permit が 1 行もなければ実質すべて拒否される |
| Q6 | D | 標準 ACL は送信元しか見ないため宛先近くに、拡張 ACL は詳細に判定できるため送信元近くに置いて早期に破棄するのが原則 |
| Q7 | A | 拡張 ACL は プロトコル → 送信元 → 宛先 → ポート条件（eq など）の順で記述する |
| Q8 | C | HTTPS は TCP/443。80 は HTTP、22 は SSH、23 は Telnet |
| Q9 | B | VTY 回線への ACL 適用は line vty コンフィグモードの `access-class <ACL> in` を使う。`ip access-group` はインターフェース用のコマンドで VTY には使えない |
| Q10 | 例 | `ip access-list extended VLAN20-TO-SRV` → `permit tcp 192.168.20.0 0.0.0.255 host 192.168.30.200 eq 80` → `deny ip any any`（明示または暗黙）。R1 の VLAN20 側インターフェース（例: Gi0/0/0.20）に `ip access-group VLAN20-TO-SRV in` を適用。拡張 ACL は詳細な条件で不要な通信を早期に破棄できるため、宛先近くではなく**送信元に近いインターフェース**に置くのが適切、という趣旨が書かれていれば正解 |

**採点**: 1 問 10 点、70 点未満は翌朝再テスト。Q10 は趣旨（送信元・宛先・プロトコル・
ポート・ワイルドカードマスク・適用方向・配置理由）が正しく含まれていれば加点する。
