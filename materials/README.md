# 教材本体（Day 1〜20）

CCNA 200-301 v1.1 準拠・20 営業日分の研修教材です。
Backlog のドキュメント機能へ転記（または Wiki API 経由で投入）して使用します。

## 構成

- `week1/`〜`week4/` — 週ごとのフォルダ（Backlog ドキュメントのフォルダ構成と対応）
- 各日 3 ファイル:
  - `dayNN-lecture.md` — 講義ドキュメント（3.5h、Backlog `01_教材` へ）
  - `dayNN-lab.md` — ラボ手順書（2.5h、Backlog `02_ラボ手順書` へ）
  - `dayNN-quiz.md` — 小テスト設問 + 解答解説（Backlog `03_小テスト` へ。
    **解答・解説セクションは講師用**。受講者へは翌日公開）

## 執筆仕様（全ファイル共通）

1. **完全オリジナル執筆**。既存教材・書籍・Web 記事の転載禁止（試験トピックの
   事実情報は可）
2. 文体は「です・ます」調の日本語。**対象は IT リテラシーゼロの社会人**
   （ペルソナは `../08-personas.md`）。専門用語・カタカナ語は初出時に地の文で
   短く正確に定義する
2b. **トーン規定**:
   - **成人学習者として扱う**。ひらがなでやわらげた表記（「きほん」等）、
     子ども向けの言い回し（「〜してみよう」「できるかな」）は使わない。
     平易さは語彙の選択と文の短さで実現する
   - たとえ話は 1 教材 2〜3 個までに厳選し、基本は実物（実際の画面・
     コマンド出力・操作）で説明する
   - 絵文字の乱用、「〜ですよね！」、励ましの連発、「イメージしてみてください」の
     多用は禁止。教科書として淡々と誠実に書く（安心させる一言は要所のみ）
3. フォーマットの基準は Day 1（`week1/day01-*.md`）に合わせる:
   - 講義: 学習目標（箇条書き）→ **ウォームアップ想起クイズ 3 問**（Day 2 以降。
     前日・3 日前・1 週間前の範囲から出題、`<details>` で解答。分散学習のため）
     → 本文（章立て・表・図解。**「💼 実務では」コラム**を 1〜2 箇所に入れ、
     試験知識と実務のつながりを示す）→ 試験のポイント引用
     → まとめ → 確認問題 5 問（`<details>` で解答）→ 次のステップ
   - ラボ: ゴール → 完成トポロジ（ASCII 図）→ 時間目安付きの手順 → 観察レポート
     設問 → 提出方法 → うまくいかないとき（表）
   - 小テスト: 設問 10 問（選択式 9 + 記述 1）→ 解答・解説の表 → 採点基準。
     週次テスト（Day 5/10/15/20）は 25 問、修了テスト（Day 20）は 60 問
4. コマンド例は Packet Tracer 9.x で実行可能な IOS コマンドのみ使用する
5. 各ファイル冒頭に配置先（Backlog ドキュメントのパス）と時間目安を引用で明記

## 制作パイプライン

教材は多段のレビュー付きパイプラインで制作:
枠組み設計 → 各日の詳細アウトライン作成 → 本文執筆 → 機械チェック（フォーマット・
技術内容・日本語）→ 指摘の修正反映。
