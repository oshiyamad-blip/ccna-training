# Day 14 小テスト: NAT（ネットワークアドレス変換）

## 設問

**Q1.** 次のうち、RFC1918 で定められた私設（プライベート）アドレスの範囲は
どれか。

- A. 172.32.0.0/12
- B. 192.168.0.0/16
- C. 200.100.0.0/16
- D. 11.0.0.0/8

**Q2.** ある内部 PC の私設アドレスが `192.168.1.10`、ルータで変換された後の
アドレスが `203.0.113.10`、通信相手の外部サーバのアドレスが `198.51.100.8`
（変換は行われない）である場合、`198.51.100.8` の呼び名として正しいものはどれか。

- A. inside local
- B. inside global
- C. outside local と outside global の両方（通常構成では同一の値になる）
- D. outside local のみで、outside global とは別の値になる

**Q3.** 少数（1〜数個）のグローバルアドレスで、数十台規模の内部ホストを
インターネットに接続させたい場合に最も適した方式はどれか。

- A. 静的 NAT
- B. 動的 NAT（プールのみ、overload なし）
- C. PAT（NAT Overload）
- D. NAT を使わずルーティングのみで対応する

**Q4.** 内部サーバ `192.168.1.100` を、常に固定のグローバルアドレス
`203.0.113.100` として外部公開したい。適切な設定コマンドはどれか。

- A. `ip nat inside source static 192.168.1.100 203.0.113.100`
- B. `ip nat inside source list 1 pool WEB overload`
- C. `ip nat outside source static 203.0.113.100 192.168.1.100`
- D. `ip nat inside source dynamic 192.168.1.100 203.0.113.100`

**Q5.** 動的 NAT を設定するために必要な要素の組み合わせとして正しいものは
どれか。

- A. スタティックルートと ACL
- B. VLAN とトランクポート
- C. 変換対象を指定する ACL と、グローバルアドレスプール
- D. HSRP のグループ番号とプライオリティ

**Q6.** `ip nat inside source list 1 interface GigabitEthernet0/1 overload`
というコマンドにおける `overload` キーワードの役割として正しいものはどれか。

- A. ACL 1 の適用範囲を無効化する
- B. ポート番号を使って複数の内部ホストを 1 つのグローバルアドレスに集約する
- C. Gi0/1 のインターフェースを inside として指定する
- D. NAT の変換エントリのタイムアウト時間を延長する

**Q7.** ルータで NAT を有効に機能させるための前提として正しいものはどれか。

- A. 内部側インターフェースに `ip nat inside`、外部側インターフェースに
  `ip nat outside` を設定する
- B. すべてのインターフェースに `ip nat outside` を設定する
- C. NAT はインターフェース設定なしで自動的に方向を判別する
- D. 内部側インターフェースにのみ `ip nat outside` を設定すればよい

**Q8.** ある NAT 構成ルータで `show ip nat translations` を実行したところ、
次の 3 行が表示された。

```
tcp 203.0.113.1:2048   192.168.1.11:2048   198.51.100.8:80   198.51.100.8:80
tcp 203.0.113.1:3050   192.168.1.12:1030   198.51.100.8:80   198.51.100.8:80
udp 203.0.113.1:4001   192.168.1.10:4001   198.51.100.4:53   198.51.100.4:53
```

この出力から判断できる変換方式として最も適切なものはどれか。

- A. 動的 NAT（プールのみ、overload なし）。複数ホストが同じプールを使っているため
- B. 静的 NAT。エントリが常時表示されているため
- C. PAT（NAT Overload）。複数の内部ホストが同一のグローバルアドレス
  `203.0.113.1` を共有し、ポート番号で区別されているため
- D. NAT が設定されておらず、ルーティングのみで通信している

**Q9.** 動的 NAT において、プールのグローバルアドレスがすべて使用中の状態で
新たな内部ホストが外部通信を試みた場合の挙動として正しいものはどれか。

- A. 最も古い変換エントリが自動的に解放され、新しいホストに割り当てられる
- B. プールの範囲外のアドレスが自動的に追加される
- C. 新規の変換が行われず、通信は失敗（ドロップ）する
- D. 自動的に PAT（overload）へ切り替わり通信は成功する

**Q10.**（記述）ある企業のオフィスに 50 台の内部端末があり、これらすべてを
1 つのグローバル IP アドレスでインターネットに接続させたい。採用すべき NAT
方式を挙げ、その理由と主要な設定コマンドを説明せよ。

## 解答・解説（翌日公開・講師用）

| 問 | 解答 | 解説 |
|---|---|---|
| Q1 | B | `192.168.0.0/16` が RFC1918 の私設アドレス。A の `172.32.0.0/12` は、私設範囲 `172.16.0.0/12`（`172.16.0.0`〜`172.31.255.255`）のすぐ外側（`172.32`〜）に位置する公開アドレスで、数字が似ているだけの定番の引っかけ。D の `11.0.0.0/8` も私設 `10.0.0.0/8` の隣にある公開アドレスで同種の罠。C の `200.100.0.0/16` も公開アドレス。私設範囲は `10/8`・`172.16/12`・`192.168/16` の 3 つのみ |
| Q2 | C | 外部ホストのアドレスが変換されない通常構成では、outside local と outside global は同一の値（198.51.100.8）になる。inside local（192.168.1.10）・inside global（203.0.113.10）は内部 PC 側の呼び名で、198.51.100.8 には該当しない |
| Q3 | C | PAT はポート番号で識別するため、少数のグローバルアドレスで多数の内部ホストを収容できる。動的 NAT（overload なし）はプールの数だけしか同時変換できない |
| Q4 | A | 静的 NAT の構文は `ip nat inside source static <inside-local> <inside-global>`。C は方向が逆、B は PAT の構文 |
| Q5 | C | 動的 NAT には変換対象を指定する ACL と、割り当て用のグローバルアドレスプールの両方が必要 |
| Q6 | B | `overload` はポート番号を使った多対 1 変換（PAT）を有効にするキーワード |
| Q7 | A | `ip nat inside` / `ip nat outside` の両方のインターフェース指定が NAT 動作の前提。片方だけや自動判別はできない |
| Q8 | C | 3 台の内部ホスト（.10/.11/.12）がいずれも同一のグローバルアドレス `203.0.113.1` を共有し、ポート番号（`:2048`/`:3050`/`:4001`）で区別されていることから PAT と判別できる。動的 NAT（overload なし）であればホストごとに別々のグローバル IP が割り当てられ、ポート番号は変換されない |
| Q9 | C | 動的 NAT ではプールが枯渇すると新規の変換ができずパケットはドロップされる。自動的に古いエントリが解放されたり PAT に切り替わったりはしない |
| Q10 | 例 | 「PAT（NAT Overload）を採用する。理由: 50 台を 1 つのグローバル IP に収容するには、IP アドレスだけでなくポート番号で内部ホストを識別する多対 1 変換が必要なため。設定例: `access-list 1 permit <内部ネットワーク> <ワイルドカードマスク>` に続けて `ip nat inside source list 1 interface <外部IF> overload`」等、PAT の選択とその理由（ポート番号による多重化）、overload を含む設定コマンドに触れていれば正解 |

**採点基準**: 1 問 10 点（満点 100 点）。70 点未満は翌朝再テストとする。Q10 は
趣旨（PAT の選択・多対 1 の仕組み・overload コマンド）が合っていれば満点。
