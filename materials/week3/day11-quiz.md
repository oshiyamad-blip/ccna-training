# Day 11 小テスト: ルーティングの基礎と静的ルート

## 設問

**Q1.** `show ip route` の出力で、ルータに直接接続されたネットワークを示す
コードはどれか。

- A. C
- B. L
- C. S
- D. O

**Q2.** 宛先 IP アドレス `172.16.10.5` に対し、ルーティングテーブルに
`172.16.0.0/16` と `172.16.10.0/24` の 2 つの経路が存在する場合、
実際に選択される経路はどれか。

- A. `172.16.0.0/16`（プレフィックス長が短いため）
- B. `172.16.10.0/24`（プレフィックス長が長いため）
- C. どちらか一方をランダムに選択する
- D. 両方とも無効になり、パケットは破棄される

**Q3.** 次のうち、Administrative Distance（AD）の値として正しい組み合わせはどれか。

- A. 直接接続 = 1、静的ルート = 0
- B. OSPF = 90、EIGRP（内部）= 110
- C. 静的ルート（ネクストホップ指定）= 1、OSPF = 110
- D. RIP = 90、EIGRP（内部）= 120

**Q4.**（応用）あるルータの `show ip route` の抜粋が次のとおりであるとき、
宛先 `172.16.5.10` 宛のパケットが実際に使用する経路はどれか。

```
S    172.16.0.0/16 [1/0] via 10.0.0.2
O    172.16.5.0/24 [110/20] via 10.0.0.6
S    172.16.5.0/24 [5/0] via 10.0.0.10
```

- A. `172.16.0.0/16` via 10.0.0.2（静的ルートで AD が最小のため）
- B. `172.16.5.0/24` via 10.0.0.6（OSPF ルートのため）
- C. `172.16.5.0/24` via 10.0.0.10（プレフィックス長が最長で、かつ AD が最小のため）
- D. 3 つの経路すべてが等コストロードバランシングで使用される

**Q5.** マルチアクセス媒体（イーサネットなど）で静的ルートを設定する際に
推奨される指定方法はどれか。

- A. 出力インターフェースのみを指定する方法
- B. ネクストホップ IP アドレスのみ、または出力インターフェースとネクストホップの
  両方を指定する完全指定
- C. AD 値のみを指定する方法
- D. メトリック値のみを指定する方法

**Q6.**（応用）R1 の Gi0/1（`192.168.10.1/30`）が、イーサネットで R2 の
`192.168.10.2` に接続されている。R2 の先にある LAN2（`192.168.20.0/24`）向けの
静的ルートを R1 に追加したい。正しい `ip route` コマンドはどれか。

- A. `ip route 192.168.20.0 255.255.255.0 192.168.10.2`
- B. `ip route 192.168.20.0/24 192.168.10.2`
- C. `ip route 255.255.255.0 192.168.20.0 192.168.10.2`
- D. `ip route 192.168.20.0 255.255.255.0 192.168.10.2 GigabitEthernet0/1`

**Q7.** フローティングスタティックルートについて正しい説明はどれか。

- A. 主経路より高い AD 値を設定し、主経路が使用できなくなったときに
  自動的にテーブルへ昇格するバックアップ経路である
- B. 主経路と同じ AD 値を設定し、常に両方の経路が同時に使われる
- C. 動的ルーティングプロトコルでのみ使用でき、静的ルートでは設定できない
- D. AD 値を 0 に固定しないと動作しない

**Q8.** IPv6 で静的ルートを機能させるために、あらかじめグローバルコンフィグ
モードで実行しておく必要があるコマンドはどれか。

- A. `ip routing`
- B. `ipv6 cef`
- C. `ip route ::/0`
- D. `ipv6 unicast-routing`

**Q9.** リンクローカルアドレス（`fe80::/10`）をネクストホップとして
IPv6 静的ルートを設定する際に必要な指定はどれか。

- A. 何も追加指定は不要で、リンクローカルアドレスだけで一意に到達できる
- B. AD 値を必ず 255 に設定する
- C. 出力インターフェースを併記する完全指定が必要（リンクローカルは
  複数リンクで重複しうるため）
- D. サブネットマスクを `/128` に固定する

**Q10.**（記述）ルータ A・B 間で静的ルートを設定する際、A から B 方向の経路
（往路）のみを設定し、B から A 方向の経路（復路）を設定しなかった場合、
なぜ A から B への ping が失敗するのかを、経路選択の観点から 1〜2 文で説明せよ。

## 解答・解説（翌日公開・講師用）

| 問 | 解答 | 解説 |
|---|---|---|
| Q1 | A | `C`（Connected）はルータに直接接続されたネットワークを表すルートコード。`L`（Local）はインターフェース自身の IP を表す `/32` ホストルート |
| Q2 | B | ロンゲストマッチの原則により、プレフィックス長がより長い（より具体的な）`172.16.10.0/24` が優先される |
| Q3 | C | 静的ルート（ネクストホップ指定）= 1、OSPF = 110 が正しい組み合わせ。直接接続 = 0、EIGRP 内部 = 90、RIP = 120 |
| Q4 | C | まずロンゲストマッチで `/16` を除外し `/24` の 2 経路に絞る。次に AD で比較し、静的ルート（AD=5）が OSPF（AD=110）より優先されるため `via 10.0.0.10` が使われる。「ロンゲストマッチ → AD → メトリック」の順の実践的な適用例 |
| Q5 | B | マルチアクセス媒体では出力インターフェースのみの指定は ARP に関する問題を招くため、ネクストホップ指定か完全指定が推奨される |
| Q6 | A | `ip route <宛先> <サブネットマスク> <ネクストホップ>` の順が正しい。B はマスクを CIDR 表記（`/24`）で書いており `ip route` の構文としては誤り、C は宛先とマスクの引数順が逆、D は完全指定の引数順が誤り（正しくは出力インターフェースの後にネクストホップ） |
| Q7 | A | フローティングスタティックは主経路より高い AD を設定し、主経路喪失時に自動的にテーブルへ昇格するバックアップ経路 |
| Q8 | D | `ipv6 unicast-routing` を有効化しないと、ルータは IPv6 パケットを転送しない。`ip routing` は IPv4 用（既定で有効） |
| Q9 | C | リンクローカルアドレスは複数リンクで重複しうるため、出力インターフェースを併記する完全指定が必須 |
| Q10 | 例 | 「B は A 行きの経路情報を持たないため、A から届いた ICMP エコーへの応答（復路のパケット）を転送できず破棄される。ルーティングは往路・復路それぞれの経路情報が独立して必要なため」等、双方向の経路が必要である点に触れていれば正解 |

**採点**: 1 問 10 点、70 点未満は翌朝再テスト。Q10 は趣旨が合っていれば 10 点。
