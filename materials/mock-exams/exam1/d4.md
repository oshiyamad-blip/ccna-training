## 設問

**Q66.** 社内PC（inside local: `192.168.1.50`）がNATルータを経由して外部サーバ `8.8.4.4` と通信しており、ルータでの変換後アドレスは `203.0.113.50` である。この構成における **inside global** に該当するアドレスはどれか。
- A. 203.0.113.50
- B. 192.168.1.50
- C. 8.8.4.4
- D. 該当するアドレスはない

**Q67.** ルータで `show ip nat translations` を実行したところ、次の出力が得られた。この出力から判断できる変換方式はどれか。

```
Pro Inside global          Inside local         Outside local        Outside global
tcp 203.0.113.5:2001       192.168.1.10:2001    8.8.8.8:80           8.8.8.8:80
tcp 203.0.113.5:2002       192.168.1.11:3050    8.8.8.8:80           8.8.8.8:80
```

- A. 静的NAT（Static NAT）
- B. 動的NAT（Dynamic NAT、プールを使用）
- C. PAT（NAT Overload）
- D. 静的PAT（ポートフォワーディング）

**Q68.** DHCPクライアントが新規にIPアドレスを取得する際、最初に送信するDiscoverメッセージの送信元アドレスと宛先アドレスの組み合わせとして正しいものはどれか。
- A. 送信元: クライアントの現在のIPアドレス、宛先: DHCPサーバのユニキャストアドレス
- B. 送信元: 0.0.0.0、宛先: 255.255.255.255
- C. 送信元: 255.255.255.255、宛先: 0.0.0.0
- D. 送信元: 169.254.0.1、宛先: DHCPサーバのユニキャストアドレス

**Q69.** 本社にあるDHCPサーバとは別サブネットにいるクライアントが、そのDHCPサーバからIPアドレスを取得できるようにしたい。クライアント側のSVIに設定すべきコマンドと、その設定が必要な理由の組み合わせとして正しいものはどれか。
- A. `ip helper-address <サーバIP>` ― DHCPのリクエストはブロードキャストであり、ルータは既定でブロードキャストを転送しないため

## 解答・解説

| 問 | 解答 | 解説（1〜2文。関連する研修Dayを明記） |
|---|---|---|
| Q66 | A | inside globalは内部端末が外部から見えるときの変換後（グローバル）アドレスであり、この例では203.0.113.50。192.168.1.50はinside local、8.8.4.4はoutside global。（→ Day14） |
| Q67 | C | 同一のinside global（203.0.113.5）に対して複数の異なるinside local（.10と.11）がポート番号で区別されて対応付けられており、これは多対1のPAT（NAT Overload）の典型的な出力。（→ Day14） |
| Q68 | B | クライアントはまだIPアドレスを持たないため、Discoverの送信元は0.0.0.0、宛先は限定ブロードキャストの255.255.255.255になる。（→ Day15） |
| Q69 | A | DHCPのDiscover/Requestはブロードキャストで送られ、ルータは既定でブロードキャストを他セグメントへ転送しないため、クライアント側SVIに`ip helper-address`を設定してユニキャストへ変換・転送させる必要がある。（→ Day15） |
| Q70 | D | MXレコードはドメイン宛のメールを処理するメールサーバを優先度付きで指定する。Aはホスト名→IPv4、CNAMEは別名、PTRは逆引き。（→ Day15） |
| Q71 | C | stratumの最大値は15で、16は有効な時刻源を持たない「未同期」を示す特別な値。（→ Day15） |
| Q72 | B | SNMPv3はユーザベースの認証と暗号化（authPriv）に対応するが、v1/v2cのコミュニティストリングは平文で送信され盗聴に弱い。ポーリングはUDP 161、Trapは162でv1/v2c/v3共通。（→ Day15） |
| Q73 | D | severityは数値が大きいほど重大度が低く、Alert=1、Critical=2、Warning=4、Notification=5であるため、この中で最も重大度が低いのはNotification。WarningとNotificationの順序は特に取り違えやすい。（→ Day15） |
| Q74 | C | SSH有効化には`hostname`→`ip domain-name`→`crypto key generate rsa`→ユーザ作成→`line vty`の設定が必要だが、NAT Overloadの設定はSSHの動作に無関係。（→ Day15） |
| Q75 | A | シェーピングは超過分をバッファに貯めて送出を遅延させ、レートを平滑化する。ポリシングは超過分を即座に破棄・再マークしバッファは行わない。（→ Day15） |
