## 解答・解説

| 問 | 解答 | 解説（1〜2文。関連する研修Dayを明記） |
|---|---|---|
| Q91 | A | SDNではコントロールプレーン（経路を決める機能）をコントローラへ集中させ、データプレーン（実際に転送する機能）は各機器に残す。（→ Day19） |
| Q92 | B | コントローラが配下機器を制御するのは南向きインターフェース（SBI）、上位のアプリや自動化ツールがコントローラを操作するのは北向きインターフェース（NBI）で、通常REST APIの形をとる。（→ Day19） |
| Q93 | C | VXLANはL2フレームをUDP（宛先ポート4789）でカプセル化し、L3ネットワーク上に論理的なL2接続を作るオーバーレイ技術（MAC-in-UDP）。TCPではなくUDPであり、アンダーレイでもトポロジ名でもない。（→ Day19） |
| Q94 | D | Assuranceは機器・クライアント・アプリケーションの健全性を可視化し問題の予兆を検知する機能。Designは設計、Policyはポリシー定義、Provisionは設定展開。（→ Day19） |
| Q95 | A | PATCHはリソースの一部だけを更新するメソッド。PUTはリソース全体の置き換え、POSTは新規作成、GETは読み取りで、部分更新には不向き。（→ Day19） |
| Q96 | B | 401は「あなたが誰か確認できない（未認証）」、403は「認証は正しいが権限が無い（未認可）」を示す。設問は認証済みだが権限が無いケースなので403が該当。（→ Day19） |
| Q97 | C | JSONのキー・値の文字列はダブルクオートで囲む必要がありシングルクオートは不可、また末尾の要素の後にカンマを付けると構文エラーになる。（→ Day19） |
| Q98 | D | Ansibleは主にSSHで対象機器へ接続するエージェントレスのツールであり、NETCONF/TCP830はSDNコントローラの南向きインターフェースなどで使われるプロトコルであってAnsibleの通信方式ではないため、この記述が誤り。（→ Day19） |
| Q99 | A | 予測AIは過去のデータパターンから将来の事象（トラフィック量、障害の予兆など）を予測するもの。文章や設定案の生成は生成AIの役割。（→ Day19） |
| Q100 | B | IBNは管理者が高レベルの「意図」を入力すると、コントローラがそれを機器ごとの具体的な設定へ変換して展開する考え方で、個々のCLI操作を管理者が意識する必要がない。（→ Day19） |
