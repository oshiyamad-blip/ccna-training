## 設問

**Q76.** サーバのOSに既知のバッファオーバーフローの欠陥が存在し、長期間パッチが適用されないまま放置されている。この欠陥そのものを指す用語として正しいものはどれか。
- A. 脆弱性（Vulnerability）
- B. 脅威（Threat）
- C. エクスプロイト（Exploit）
- D. 緩和策（Mitigation）

**Q77.** 大量のボットネット端末から標的のWebサーバへ同時にトラフィックが送り込まれ、サーバのリソースが枯渇して正規利用者がサービスにアクセスできなくなった。この攻撃が直接的に侵害したCIAトライアドの要素はどれか。
- A. 機密性（Confidentiality）
- B. 可用性（Availability）
- C. 完全性（Integrity）
- D. 真正性（Authenticity）

**Q78.** 攻撃者が電話でユーザーに連絡し、社内の情報システム部門の担当者を装って「アカウントロック解除のため」と称し、口頭でパスワードを聞き出した。この手口に最も近い用語はどれか。

**Q79.** Cisco機器の管理者アカウントについて、機器ごとに実行できるコマンドを細かく制御したい。この要件に最も適したAAAプロトコルと、その理由の組み合わせとして正しいものはどれか。
- A. TACACS+ ― 認証・認可・アカウンティングを分離して扱え、コマンド単位の認可を柔軟に設定できるため
- B. TACACS+ ― UDP 1812/1813を使用し、パスワード部分のみを暗号化するため

**Q80.** ログイン時にスマートフォンの認証アプリが表示するワンタイムコードの入力と、指紋認証の両方を要求する構成がある。この構成に関する説明として正しいものはどれか。

**Q81.** ルータに`enable secret`と`enable password`の両方が設定されている状態で`service password-encryption`を有効化した。この状況に関する記述として正しいものはどれか。
- A. `enable password`が優先され、`service password-encryption`はrunning-config内の平文パスワードをMD5でハッシュ化する

**Q82.** ルータに`login block-for 60 attempts 3 within 30`を設定した。この設定の動作として正しいものはどれか。

**Q84.** 次のACLを適用したところ、192.168.10.5からの通信も許可されてしまい、意図した拒否が機能しなかった。原因として正しいものはどれか。

```
access-list 10 permit 192.168.10.0 0.0.0.255
access-list 10 deny host 192.168.10.5
```

- A. 標準ACLでは `deny host` という書式を使用できないため
- B. permit行とdeny行の順序が逆であり、より広い条件のpermit行が先に一致してしまい、deny行が一切評価されないため
- C. 暗黙のdeny anyが常にpermitより優先して評価されるため
- D. このACLはout方向ではなくin方向に適用する必要があるため

**Q85.** 経理部門（192.168.10.0/24）からファイルサーバ（192.168.30.100）への通信のみを拒否し、経理部門から他の宛先への通信には影響を与えたくない。この要件を満たすACLの設計として最も適切なものはどれか。

## 解答・解説

| 問 | 解答 | 解説（1〜2文。関連する研修Dayを明記） |
|---|---|---|
| Q76 | A | 「システムに存在する弱点・欠陥」は脆弱性。脅威はそれを突く危険そのもの、エクスプロイトは突くための手段・コード、緩和策は対処。（→ Day16） |
| Q77 | B | DoS/DDoSはサーバやネットワークのリソースを枯渇させ正規利用者がサービスを使えなくする攻撃であり、CIAトライアドの可用性を直接狙う。（→ Day16） |
| Q78 | D | 電話（Voice）を使い、担当者になりすまして情報を聞き出す手口はビッシング。スミッシングはSMS、テールゲーティングは物理的な入室追随、ホエーリングは経営層を狙うスピアフィッシング。（→ Day16） |
| Q79 | A | TACACS+はTCP 49でパケット全体を暗号化し、認証・認可・アカウンティングを分離できるためコマンド単位の柔軟な認可に強い。RADIUSはUDPでパスワード部分のみ暗号化し、認証と認可を統合して扱う。（→ Day16） |
| Q80 | C | MFAは「異なる種類」の要素を2つ以上組み合わせて成立する。認証アプリは「所持」、指紋は「生体」要素であり異なる種類なのでMFAに該当する。数ではなく種類が重要。（→ Day16） |
| Q81 | B | 両方設定されている場合は`enable secret`が優先される。`service password-encryption`はType 7という可逆的な弱い難読化であり、専用ツールで容易に復号できるため本質的な保護にはならない。（→ Day16） |
| Q82 | D | `login block-for 60 attempts 3 within 30`は、30秒以内に3回ログインに失敗すると60秒間すべてのログイン試行をブロックする設定で、ブルートフォース攻撃の緩和が目的。（→ Day16） |
| Q83 | A | /22のサブネットマスクは255.255.252.0であり、ワイルドカードマスクは255.255.255.255から引いた0.0.3.255。第3オクテットの3は下位2ビット（4つの/24）が任意であることを意味する。（→ Day17） |
| Q84 | B | ACLは上から順に評価され最初に一致したACEで処理が確定する。より広い条件のpermitを先に置くと、後続のより限定的なdenyが一切ヒットしなくなる。限定的な条件を上に置く必要がある。（→ Day17） |
| Q85 | D | 標準ACLは送信元しか見ないため経理部門の送信元近くに置くと他の宛先への通信まで巻き込む。宛先まで条件にできる拡張ACLを使い、不要な通信を早期に破棄する原則どおり送信元近くに適用するのが正しい。（→ Day17） |
| Q86 | B | VTY回線へのアクセス制限にはインタフェース用の`ip access-group`ではなく、回線専用の`access-class <ACL> in`を使う。この違いは頻出の落とし穴。（→ Day17） |
| Q87 | A | protectモードはフレームを破棄しつつポートの稼働を継続し、Syslog通知も違反カウンタの増加も行わない唯一のモード。restrictは通知とカウンタ増加あり、shutdown（既定）はerr-disabledに遷移する。（→ Day18） |
| Q88 | C | untrustedポートで破棄されるのは、DHCPサーバが送るはずのメッセージ（DHCPOFFER/DHCPACK/DHCPNAK）。クライアント発のDISCOVER/REQUESTは通常どおり転送される。（→ Day18） |
| Q89 | C | DAIはDHCPスヌーピングのバインディングテーブル（IP・MAC・ポートの対応情報）を判定根拠にするため、DHCPスヌーピングが先に有効化され機能している必要がある。（→ Day18） |
| Q90 | C | サイト間VPNはゲートウェイ（ルータ/FW）同士でIPsecトンネルを確立しネットワーク全体を透過的に接続する方式で、端末側にクライアントソフトは不要。リモートアクセスVPNは個々の端末にクライアントソフトが必要。（→ Day18） |
