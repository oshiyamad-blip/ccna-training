## 設問

## 解答・解説

| 問 | 解答 | 解説(1〜2文。関連する研修Dayを明記) |
|---|---|---|
| Q21 | B | VLANは1台の物理スイッチを論理的に分割し、VLANごとに独立したブロードキャストドメインを作る仕組み。(→ Day6) |
| Q22 | D | VLAN番号・名前などの情報は `vlan.dat` としてフラッシュに保存され、running-config/startup-configとは別管理。`no vlan` で削除すると割り当て済みポートはinactiveになる。(→ Day6) |
| Q23 | A | `switchport access vlan` で未作成のVLANを指定すると、IOSが自動的にそのVLANを作成する(名前は自動生成名になる)。(→ Day6) |
| Q24 | C | 802.1Qタグは4バイトで、送信元MACアドレスとEtherTypeの間に挿入される。VLAN IDは12ビット。(→ Day7) |
| Q25 | B | ネイティブVLAN不一致はCDPにより `%CDP-4-NATIVE_VLAN_MISMATCH` として検出されることがある。ネイティブVLANのフレームはタグなしで送信される。(→ Day7) |
| Q26 | A | `dynamic auto` 同士はどちらも相手からの働きかけを待つ受け身のため、トランクは成立せずアクセスポートのままとなる。(→ Day7) |
| Q27 | D | `add` を付けずに `allowed vlan` を実行すると既存リストが丸ごと上書きされてしまうため、追加時は必ず `add` を使う。(→ Day7) |
| Q28 | C | サブインタフェースでは先に `encapsulation dot1q <VLAN-ID>` でVLANを紐付けてから `ip address` を設定する必要がある。(→ Day8) |
| Q29 | A | L3スイッチは初期状態でL2スイッチとしてしか動作せず、`ip routing` を有効化しないとVLAN間ルーティングが行われない。(→ Day8) |
| Q30 | D | SVIがup/upになるにはVLANの存在・所属ポートのup・no shutdownの3条件が必要で、`ip routing` はVLAN間通信自体の前提として有効化されている必要がある(無効化されていてはならない)。(→ Day8) |
| Q31 | D | ルートブリッジはBID(ブリッジプライオリティ+MACアドレス)が最小のスイッチが選出され、同値の場合はMACアドレスが最小のスイッチが選ばれる。(→ Day9) |
| Q32 | C | ブリッジプライオリティは4096の倍数(0〜61440)でのみ設定可能。(→ Day9) |
| Q33 | B | 直接リンク障害はリンクダウンを即座に検知できるため、Max Ageを待たず2×Forward Delay(15秒×2=30秒)で収束する。(→ Day9) |
| Q34 | A | バックアップポートは指定ポートのバックアップとして機能し、代替ポートはルートポートのバックアップとして機能する。(→ Day9) |
| Q35 | D | BPDU Guardは、PortFastを設定したポートがBPDUを受信した場合にそのポートをerr-disable状態にして強制的に無効化する。(→ Day9) |
| Q36 | C | LACPの `active`-`passive` の組み合わせはトランク(EtherChannel)が成立する。成立しないのは `passive`-`passive` の組み合わせのみ。(→ Day9) |
| Q37 | B | 無線LANは衝突検出が技術的に困難なため、衝突を未然に回避するCSMA/CAを採用する(有線はCSMA/CD)。(→ Day10) |
| Q38 | A | 2.4GHz帯でチャネル間隔が重ならず干渉しない組み合わせは1・6・11のみ。(→ Day10) |
| Q39 | D | CAPWAPの制御トンネルはUDP 5246、データトンネルはUDP 5247を使用する。(→ Day10) |
| Q40 | C | LLDPはIEEE 802.1ABで標準化されたオープンプロトコルでCisco機器では既定で無効(`lldp run` で有効化)。CDPはCisco独自でアドバタイズ間隔60秒/ホールドタイム180秒が既定。(→ Day10) |
