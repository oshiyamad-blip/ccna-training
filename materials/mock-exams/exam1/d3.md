## 解答・解説

| 問 | 解答 | 解説(1〜2文。関連する研修Dayを明記) |
|---|---|---|
| Q41 | C | インタフェースにIPを設定しupすると、そのサブネット全体を示すC(connected)経路と、自身のIPを示す/32のL(local)経路が自動生成される。(→ Day11) |
| Q42 | A | ロンゲストマッチの原則により、宛先に一致する経路のうちプレフィックス長が最も長い(最も具体的な)`/24`が優先される。(→ Day11) |
| Q43 | D | AD値は静的ルートが1、EIGRP(内部)が90であり、値が小さいほど優先されるため静的ルートが選ばれる。(→ Day11) |
| Q44 | B | イーサネットのようなマルチアクセス媒体で出力インタフェースのみを指定するとネクストホップが特定できず、プロキシARPに依存した非効率な処理を招くため非推奨。(→ Day11) |
| Q45 | C | フローティングスタティックは主経路(この場合OSPFのAD110)より大きいAD値をコマンド末尾に指定することで、平常時はテーブルに載らないバックアップ経路として機能させる。(→ Day11) |
| Q46 | A | `0.0.0.0/0`はプレフィックス長が0で最も緩い一致のため、より具体的な経路が存在すればロンゲストマッチにより必ずそちらが優先される。(→ Day11) |
| Q47 | D | IPv4の`ip routing`と異なり、IPv6では`ipv6 unicast-routing`を明示的に有効化しない限りルータはIPv6パケットを転送しない。(→ Day11) |
| Q48 | B | リンクローカルアドレスは各リンクごとに重複しうるため、`ipv6 route`でリンクローカルをネクストホップに使う場合は出力インタフェースの併記(完全指定)が必須。(→ Day11) |
| Q49 | A | Router-on-a-Stickの全サブインタフェースは1本の物理トランクリンクの上に成り立っているため、物理IFがdownすると全VLANの通信が同時に停止する。(→ Day8) |
| Q50 | C | 機器同士のpoint-to-pointなL3直結リンクには、物理ポートに`no switchport`を設定して直接IPを付与するルーテッドポートを使うのが定石。(→ Day8) |
| Q51 | B | OSPFはLSAを交換して同一のLSDB(地図)を全ルータで共有し、各自がSPF(ダイクストラ)で最短経路を計算するリンクステート方式である。(→ Day12) |
| Q52 | D | AD値は直接接続0・静的1・EIGRP(内部)90・OSPF110・RIP120の順で、数値が小さいほど信頼度が高く優先される。(→ Day12) |
| Q53 | C | Router IDはネイバー成立の一致条件ではなく、ドメイン内で重複しない一意性が求められるのみ。エリアID・タイマー・サブネットは一致が必須。(→ Day12) |
| Q54 | A | マルチアクセス網ではDR/BDRに情報を集約する設計のため、DROTHER同士は2-Wayで停止するのが正常な動作であり異常ではない。(→ Day12) |
| Q55 | D | Router IDは明示指定→稼働中の最大ループバックIP→稼働中の最大物理IPの順で決まり、ループバック未設定なら物理インタフェースの最大IPが採用される。(→ Day12) |
| Q56 | B | DR/BDRの選出は非プリエンプティブであり、後からより高いプライオリティのルータが参加しても既存のDRは交代しない。(→ Day12) |
| Q57 | A | 既定の参照帯域幅10^8(100Mbps)ではGigabitEthernetのコストも切り上げで1となり、FastEthernetと同じコストになってしまう。(→ Day12) |
| Q58 | C | `/26`のサブネットマスクは255.255.255.192であり、255.255.255.255から引き算するとワイルドカードマスクは0.0.0.63になる。(→ Day12) |
| Q59 | D | 2-WayまではMTUに関係なく進むが、DBD交換(Exstart/Exchange)で双方のMTUが食い違っているとそれ以降へ進めず停滞する。(→ Day12・Day13) |
| Q60 | B | passive-interfaceを設定したインタフェースはHelloパケットを送出しなくなるため、そのリンク経由ではネイバーが一切形成されない。(→ Day13) |
| Q61 | D | `default-information originate`単体では自身にデフォルトルートが存在しないと広告されないため、無条件で広告するには`always`キーワードが必要。(→ Day13) |
| Q62 | A | E2(既定のメトリックタイプ)は広告時に設定されたメトリック値が固定され、途中のルータを何台経由しても変化しない(累積加算されるのはE1)。(→ Day13) |
| Q63 | B | FHRPは複数台のルータで仮想IP・仮想MACを共有し、デフォルトゲートウェイが1台に固定されることによる単一障害点を排除する仕組み。(→ Day13) |
| Q64 | C | GLBPはAVG(Active Virtual Gateway)が複数のAVF(Active Virtual Forwarder)に負荷を分散させる点が、Active/Standbyの1台構成であるHSRP・VRRPと異なる。(→ Day13) |
| Q65 | D | HSRPのプライオリティ既定値は100で、Active選出後に高プライオリティのルータが復旧しても自動的には奪い返さないよう、プリエンプトは既定で無効になっている。(→ Day13) |
