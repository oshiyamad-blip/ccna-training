## 設問

**Q1.** ネットワーク機器と、フレーム／パケットの転送先を決定する際に参照するアドレスの組み合わせとして正しいものはどれか。

- B. L2スイッチ — IPアドレス

- D. ハブ — IPアドレス

**Q2.** カプセル化の過程で、トランスポート層のセグメントにIPヘッダを付加して生成されるPDUとして正しいものはどれか。
- A. フレームにIPヘッダを付加してパケットにする
- B. セグメントにIPヘッダを付加してパケットにする
- C. パケットにEthernetヘッダを付加してフレームにする
- D. セグメントにEthernetヘッダを付加してフレームにする

**Q3.** データセンターで採用されるSpine-Leafトポロジの利点として最も適切なものはどれか。

- D. ポートごとにブロードキャストドメインを分割できる

**Q4.** ルータのコンソールポートに端末エミュレータで接続する際の既定のシリアル設定として正しいものはどれか。
- A. 115200 bps / 8-N-1

**Q6.** `Router(config-if)#` の状態でコマンド `end` を実行すると、どのモードへ移行するか。

**Q7.** ルータでSSHを有効化するための前提設定として、正しい順序はどれか。

**Q8.** L2スイッチの管理用IPアドレスの設定方法、および別サブネットから管理する場合の要件として正しいものはどれか。

## 解答・解説

| 問 | 解答 | 解説（1〜2文。関連する研修Dayを明記） |
|---|---|---|
| Q1 | A | L2スイッチはMACアドレス（L2）で転送先を判断し、ルータはIPアドレス（L3）で判断する。（→ Day1） |
| Q2 | B | L4のセグメントにIPヘッダを付加したものがL3のパケットであり、フレーム化はさらに下のL2の処理。（→ Day1） |
| Q3 | C | Spine-Leafはどの Leaf 同士も Spine を1台挟んだ等距離になるため、East-West通信の遅延が予測しやすい。Aは実際の構成と異なり、Bは2階層構成の説明。（→ Day1） |
| Q4 | D | コンソール接続の既定値は「9600 / 8-N-1・フロー制御なし」。（→ Day2） |
| Q5 | A | `enable secret` はハッシュ化して保存され、`enable password`（平文）より常に優先される。（→ Day2） |
| Q6 | B | `end`（または Ctrl+Z）はどのモードにいても一気に特権EXECモードへ戻る。1つ上へ戻るのは `exit`。（→ Day2） |
| Q7 | C | RSA鍵の名前にホスト名とドメイン名が使われるため、`hostname` → `ip domain-name` → `crypto key generate rsa` → ローカルユーザ の順が必須。（→ Day2） |
| Q8 | D | スイッチはSVIに管理IPを設定し、L2スイッチはルーティング機能を持たないため別サブネットから管理する場合は `ip default-gateway` が必要。（→ Day2） |
| Q9 | A | マスク255.255.255.192（/26）のブロックサイズは64。`.90` は `.64`〜`.127` の範囲に属するため、ネットワークアドレスは172.16.50.64。（→ Day3） |
| Q10 | B | 2^5−2=30≥20を満たす最小のホスト部ビット数は5のため、プレフィックス長は32−5=/27。/26では無駄が多い。（→ Day3） |
| Q11 | C | RFC1918の私設範囲は172.16.0.0〜172.31.255.255であり、172.32.5.5はこの範囲を外れるグローバルアドレス。（→ Day3） |
| Q12 | D | DHCPでアドレスを取得できなかった端末はAPIPA（169.254.0.0/16）を自動的に割り当てる。（→ Day3） |
| Q13 | A | ルータ間のP2Pリンクは2台分のアドレスしか不要なため、/30または/31が最も無駄が少ない。（→ Day3） |
| Q14 | B | 先頭ゼロの省略と連続ゼロブロックの`::`圧縮（1回のみ）を適用すると `2001:db8::1` になる。（→ Day4） |
| Q15 | C | リンクローカルアドレスは `fe80::/10` の範囲で、IPv6有効インタフェースに必ず自動生成される。（→ Day4） |
| Q16 | D | MACアドレスの前半/後半の間に `fffe` を挿入し、先頭バイトのU/Lビットを反転（00→02）すると `0250:56ff:feaa:bbcc` になる。（→ Day4） |
| Q17 | A | SLAACではホストがRouter Solicitationをマルチキャストで送信し、ルータがRouter Advertisementでプレフィックスを返す。（→ Day4） |
| Q18 | B | TCPはSYN→SYN/ACK→ACKの3ウェイハンドシェイクでセッションを確立するが、コネクションレス型のUDPは確立しない。（→ Day5） |
| Q19 | C | 宛先MACが不明（unknown unicast）な場合、スイッチは受信ポート以外の全ポートへフラッディングする。（→ Day5） |
| Q20 | D | デュプレックスミスマッチでは、autoにフォールバックした半二重側でlate collisionが、固定側の全二重でCRC/FCSエラーが増加する。リンク自体はupのままダウンしない。（→ Day5） |
