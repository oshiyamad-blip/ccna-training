# 模試の設計方針と本試験類似度チェック基準

## 本試験（CCNA 200-301 v1.1）の実際の形式（2026年時点の調査）

| 項目 | 本試験 |
|---|---|
| 問題数 | 約 100〜120 問（受験報告では 88〜105 問など変動あり） |
| 制限時間 | 120 分 |
| 合格ライン | 825 / 1000 前後（非公開） |
| 設問形式 | ①単一選択（4択） ②**複数選択**（Choose two/three） ③**ドラッグ＆ドロップ**（対応付け・並べ替え） ④**シミュレーション/実技**（仮想IOS環境での設定・トラブルシュート） |
| ドメイン比率 | 基礎20% / アクセス20% / IPコネクティビティ25% / IPサービス10% / セキュリティ15% / 自動化10% |

## 本模試の設計と本試験との対応

| 本試験の要素 | 本模試での扱い |
|---|---|
| 単一選択 | そのまま出題（主形式） |
| 複数選択（Choose two 等） | 各回に一定割合で含める（類似度チェックで検証・調整） |
| ドラッグ＆ドロップ | Backlog のテキスト運用では再現不可のため、**対応付け問題（A〜Dの組み合わせ選択）・手順並べ替え問題**として形式変換して出題 |
| シミュレーション/実技 | 模試とは別枠で、試験対策フェーズの **Packet Tracer 実技ミニタスク**（Day22/24 の総仕上げラボ）が対応。模試当日の午後にも小型の実技課題を組み合わせる。加えて、各模試の末尾に **「シムレット演習（実技代替・別枠）」** を 1 本用意（show 出力・構成を読み解く S1〜S4 の多段設問）。本試験の Sim/Simlet で問われる「出力解釈 → 切り分け → 対処 → 結果予測」を紙上で練習する。本編 100 問の採点には含めない |
| 合格ライン 825/1000 | 受験 GO 基準は余裕を持たせて「総合 85% 以上を直近 2 回連続（うち 1 回以上は重い回 ②・④・⑤ で達成）」（正式版は `07-exam-phase.md`）。①は肩慣らし用の易しめの回で単独判定には使わない |

## 類似度チェックの観点（DEV-17）

1. **形式の内訳**: 単一選択だけに偏っていないか（複数選択・対応付け・並べ替えの比率）
2. **出題スタイル**: 単純な知識想起だけでなく、show 出力・設定断片・トポロジ想定の
   シナリオ問題が本試験相当の割合（体感 4〜5 割）あるか
3. **難易度分布**: 即答できる問題〜2 分かかる問題のバランス。全問が易しすぎ/難しすぎないか
4. **ドメイン比率**: 公式比率どおりか（設問番号の割当で担保済み・要再確認）
5. **設問文の作法**: 本試験特有の言い回し（「最も適切なものはどれか」「2つ選べ」、
   否定形問題の少なさ）に沿っているか
6. **時間設計**: 100 問 120 分で解き切れる文章量か

チェックは模試完成後に高能力モデルの独立レビューで実施し、乖離があれば修正する。
