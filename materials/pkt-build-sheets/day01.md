## 2. 結線表

| 機器A・ポート | ケーブル種別 | 機器B・ポート |
|---|---|---|
| PC0 / FastEthernet0 | ストレートケーブル | Switch0 / FastEthernet0/1 |
| PC1 / FastEthernet0 | ストレートケーブル | Switch0 / FastEthernet0/2 |

- ケーブルは「自動選択（Automatically Choose Connection Type）」で可（PC-スイッチ間はストレート判定になる）。もし赤くなった場合は上表のとおり明示的に「ストレートケーブル」を選び直す。
- 配線後、Switch0 起動直後はポートが橙 → 約30秒でSTP収束し緑になる。**緑になるまで待ってから保存する。**

## 4. 貼り付け用コンフィグ（事前設定）

**事前設定なし（機器は初期状態）。**

- Switch0: ホスト名すら未設定の工場出荷状態のまま配置・保存する（IOS には一切コマンドを入力しない）。
- PC0 / PC1: 上記のとおり IP アドレスも未設定のまま保存する。

これはレベル B の定義（機器配置 + 全ケーブルのみ）どおりであり、IP 設計・設定自体が本日の学習対象であるため。

## 5. 完成コンフィグ（`day01_answer.pkt` 用・講師用）

このラボは IOS への CLI 設定コマンド入力が一切ない（スイッチは L2 のまま何も設定しない）。完成見本として用意すべきなのは **PC の IP Configuration** のみ。

### PC0（Desktop > IP Configuration）

```
IP Address:      192.168.1.10
Subnet Mask:     255.255.255.0
Default Gateway: (空欄のまま)
```

### PC1（Desktop > IP Configuration）

```
IP Address:      192.168.1.20
Subnet Mask:     255.255.255.0
Default Gateway: (空欄のまま)
```

### Switch0

```
! コンフィグ投入なし。工場出荷時の初期状態のまま。
! (hostname 変更やVLAN設定などは本ラボの範囲外)
```

### 確認用コマンド（PC0 の Command Prompt、講師が動作確認する際の参考）

```
ipconfig
ping 192.168.1.20
arp -a
```

- 1 回目の ping は最初の 1〜2 発がタイムアウトする場合がある（ARP 解決待ちのため正常）。2 回目の実行で全成功することを確認する。
