## 4. 貼り付け用コンフィグ（事前設定）

事前設定なし（機器は初期状態）。

day00 は「配置・配線・IP設定・ping」そのものが学習対象であり、`pkt-build-spec.md` の作り込みレベル定義に沿えば、開始ファイルを配布する場合でも Switch0 に対する事前の CLI 投入は一切行わない（ホスト名すら未設定の工場出荷状態）。

## 5. 完成コンフィグ（`day00_answer.pkt` 用・講師用）

本ラボは IOS コンフィグの投入が学習対象ではないため、動作（PC0-PC1間 ping 成功）に**必須の CLI コマンドは存在しない**。Catalyst 2960 は工場出荷状態のまま、同一 VLAN（デフォルト VLAN 1）の2つのアクセスポート間でフレームを転送できるため、ケーブル接続と PC の IP 設定だけで ping は成功する。

参考として、質問対応時に「スイッチにも設定はできる」と示すための任意コマンド例（**必須ではない**）:

```
enable
configure terminal
hostname Switch0
end
write memory
```

- Switch0 に対して行うのはホスト名変更のみ（任意）。VLAN・トランク・IPなど day00 で扱わない設定は一切投入しない。
- PC0 / PC1 は CLI ではなく Desktop > IP Configuration で設定するため、コンフィグ貼り付けは不要（上記セクション3の値をGUIに入力するのみ）。

## 6. 組み立て後チェック

- [ ] PC0-Switch0間、PC1-Switch0間のリンクが両方とも緑（オレンジ点滅が収まるまで待つ）
- [ ] PC0: IP `192.168.1.10` / 255.255.255.0、PC1: IP `192.168.1.20` / 255.255.255.0 がそれぞれ Desktop > IP Configuration に設定されている
- [ ] Switch0 に不要な設定（VLAN・IP・ホスト名以外の何か）が入っていない＝初期状態に近いこと（本題が未設定＝そもそも本題がないラボなので「何も設定していない」ことを確認）
- [ ] PC0 の Command Prompt から `ping 192.168.1.20` を実行し、`Reply from 192.168.1.20 ...` が返る（最初の1〜2行が `Request timed out.` になるのは正常。再実行で全成功すればOK）
- [ ] PC1 側からも `ping 192.168.1.10` が成功する（逆方向も確認）
- [ ] `File > Save As` で `day00_answer.pkt`（または配布する場合は `day00_start.pkt`）として保存し、再度開いて配置・配線・IP設定が保持されていることを確認
