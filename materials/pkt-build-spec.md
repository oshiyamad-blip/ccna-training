# ラボ開始用 .pkt ファイル 作成仕様書

各ラボの「開始用 Packet Tracer ファイル（`.pkt`）」を人が作成するための仕様です。
Packet Tracer は GUI 操作のため AI では作れません。この仕様に沿って一度作れば、
以後の開講で流用できます（初回だけの作業）。

> 前提バージョン: Cisco Packet Tracer 9.x。保存形式は `.pkt`。
> 各ラボの**正確な機器・ポート・IP は、そのラボ手順書（`week*/dayNN-lab.md`）の
> 「完成トポロジ」図と IP アドレス表が唯一の正**です。本仕様は「開始ファイルに
> どこまで作り込むか（＝受講者に何を残すか）」を定義します。

## 基本方針 — 「その日に学ぶ設定」だけを受講者に残す

開始ファイルは、**その日のテーマである IOS 設定を受講者が入力する直前の状態**に
します。それ以外（機器の配置・配線・PC の IP など、その日の学習対象でない部分）は
作り込んでおき、受講者が本題に集中できるようにします。これはシミュレーションで
判明した「未経験者は本題に入る前の準備で時間と自信を消耗する」問題への対策です。

作り込みレベルを 3 段階で指定します。

| レベル | 開始ファイルに含めるもの | 受講者が行うこと |
|---|---|---|
| **A（配線済み・IP済み）** | 機器配置 + 全ケーブル + PC/サーバの IP・GW 設定済み。ルータ/スイッチは**未設定**（ホスト名すら未設定の初期状態） | その日のテーマの IOS 設定 |
| **B（配線済み・IP未）** | 機器配置 + 全ケーブルのみ。PC の IP も受講者が設定 | IP 設計・設定 + IOS 設定（IP 設計自体が学習対象の日） |
| **C（前提設定済み）** | レベル A に加え、**前日までの設定や、その日の“前半の完成状態”を投入済み** | その日の後半・応用部分のみ（障害切り分け系・積み上げ系の日） |

共通ルール:

- 機器モデルはラボ手順書の指定に合わせる（例: ルータ ISR 4331/2911、スイッチ 2960、
  L3 は 3560）。手順書のコマンド（インターフェース名 Gi0/0 等）と機器の
  インターフェース構成が一致するモデルを選ぶ
- ケーブルは「自動選択（Automatically Choose Connection Type）」で可。ただし
  リンクが赤くなる場合は手順書指定の種別（ストレート/クロス）で結線
- リンクが**緑**になる（STP 収束済み）状態まで待ってから保存する
- ファイル名は `dayNN_start.pkt`（例: `day06_start.pkt`）。Day0/P5 用は `day00_start.pkt`
- 保存前に受講者視点で一度開き、意図した状態（本題が未設定）になっているか確認

## 検収チェック（各ファイル共通）

- [ ] 手順書のトポロジ図・IP 表と機器/ポート/IP が一致
- [ ] 指定した作り込みレベルどおり（本題が未設定で残っている）
- [ ] 全リンクが緑
- [ ] PC から（IP 済みレベルなら）同一セグメント内 ping が通る
- [ ] `.pkt` で保存し、再度開いて状態が保たれる

## 作成の進め方（推奨）

1. まず **Day1〜5** を作る（Week1 が始まればすぐ必要）。1個15〜30分が目安
2. 障害シナリオ入り（Day13）と総合演習（Day20）は時間がかかるので優先的に着手
3. 各ファイルは完成後、上の「検収チェック」を実施
4. 作成した `.pkt` は各ラボ課題に添付する（`upload-wiki.mjs` は Markdown 教材専用の
   ため、`.pkt` は Backlog 課題への手動添付、またはファイル共有で配布する）
5. 完成見本（`_answer.pkt`）は講師用フォルダに保管し、質問対応・採点に使う
