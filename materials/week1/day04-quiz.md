## 設問

**Q1.** `2001:0db8:0000:0000:0000:0000:0000:0001` を正しく省略した表記はどれか。

- A. `2001:db8::1`
- B. `2001:db8::2`
- C. `2001:dba::1`
- D. `2001:db8::1::0`

**Q2.** 次のうち、`::` の使用ルールに違反しており無効なアドレス表記はどれか。

- A. `2001:db8::1`
- B. `2001:0:0:1::2:1::3`
- C. `fe80::1`
- D. `2001:db8:0:1::a:1`

**Q3.** 次のうち、グローバルユニキャストアドレス（GUA）に分類されるものはどれか。

- A. `fe80::1234`
- B. `fc00::10`
- C. `2001:db8:acad::10`
- D. `ff02::1`

**Q4.** 次のうち、同一リンク内でのみ有効で、必ず自動的に生成されるアドレスの範囲はどれか。

- A. `2000::/3`
- B. `fc00::/7`
- C. `ff00::/8`
- D. `fe80::/10`

**Q5.** アドレス自体は SLAAC で自動生成し、DNS サーバなどの補足情報だけを DHCPv6
サーバから取得する方式はどれか。

- A. ステートフル DHCPv6
- B. ステートレス DHCPv6
- C. SLAAC 単独（DHCPv6 は使用しない）
- D. 静的アドレス設定

**Q6.** IPv4 の ARP に相当し、Neighbor Solicitation / Neighbor Advertisement の
やり取りでアドレス解決を行い、重複アドレス検出（DAD）にも使われるプロトコルは
どれか。

- A. NDP（近隣探索プロトコル）
- B. DHCPv6
- C. SLAAC
- D. ARP

**Q7.** MAC アドレス `0011.2233.4455` を EUI-64 方式でインターフェース ID に
変換した結果として正しいものはどれか。

- A. `0011:22ff:fe33:4455`
- B. `0211:2233:fffe:4455`
- C. `0211:22ff:fe33:4455`
- D. `0011:22fe:ff33:4455`

**Q8.** SLAAC（ステートレスアドレス自動設定）の説明として正しいものはどれか。

- A. ホストが DHCPv6 サーバからアドレス・DNS などすべての情報を取得する方式である
- B. ルータが ARP 要求を送信してホストの MAC アドレスを学習する方式である
- C. ホストが静的にアドレスを入力し、ルータに登録する方式である
- D. ホストが RA から得たプレフィックスと自身で生成したインターフェース ID を
  組み合わせてアドレスを構成する方式である

**Q9.** ルータで異なるセグメント間の IPv6 パケットを転送できるようにするために、
グローバルコンフィギュレーションモードで必ず入力する必要があるコマンドはどれか。

- A. `ip routing`
- B. `ipv6 enable`
- C. `ipv6 unicast-routing`
- D. `ipv6 route 0::/0`

**Q10.**（記述）ルータでの IPv6 静的設定手順（`ipv6 unicast-routing` と
`ipv6 address` コマンド）、または SLAAC でホストがアドレスを得る一連の動作の
どちらか一方を選び、具体的な手順やコマンドを交えて説明せよ。

## 解答・解説（翌日公開・講師用）

| 問 | 解答 | 解説 |
|---|---|---|
| Q1 | A | 先頭ゼロを省略し、連続する 0 ブロックを `::` で 1 回だけ圧縮すると `2001:db8::1` になる |
| Q2 | B | `::` はアドレス中に 1 回しか使用できない。2 回使うと復元位置が一意に決まらず無効になる |
| Q3 | C | GUA は `2000::/3`（先頭が `2` または `3`）の範囲。A は LLA、B は ULA、D はマルチキャスト |
| Q4 | D | `fe80::/10` はリンクローカルアドレスの範囲で、IPv6 有効なインターフェースに必ず自動生成される |
| Q5 | B | ステートレス DHCPv6 はアドレス自体を SLAAC で取得し、DNS サーバなどの補足情報だけを DHCPv6 サーバから取得する方式。すべてを DHCPv6 サーバから取得するのはステートフル DHCPv6（A） |
| Q6 | A | NDP（近隣探索プロトコル）は ICMPv6 を使い、Neighbor Solicitation/Advertisement によるアドレス解決や DAD を行う、IPv4 の ARP（D）に相当するプロトコル |
| Q7 | C | MAC を前半/後半に分割し中央に `fffe` を挿入、先頭バイト `00` の U/L ビットを反転させると `02` になり `0211:22ff:fe33:4455` となる |
| Q8 | D | SLAAC はホストが RS/RA（NDP）でプレフィックスを取得し、自身でインターフェース ID を生成してアドレスを組み立てる方式。A はステートフル DHCPv6 の説明 |
| Q9 | C | `ipv6 unicast-routing` を入力しない限り、ルータは IPv6 ホストとして動作するのみで異なるセグメント間のパケットを中継しない |
| Q10 | 例 | 静的設定なら「グローバルで `ipv6 unicast-routing` を有効化 → インターフェースで `ipv6 address <アドレス>/64` → `no shutdown`」の流れ、SLAAC なら「ホストが RS をマルチキャスト送信 → ルータが RA でプレフィックスを返す → ホストが受け取ったプレフィックスと自身で生成したインターフェース ID を組み合わせてアドレスを構成する」という NDP を用いた一連の動作に触れていれば正解 |

**採点**: 1 問 10 点、70 点未満は翌朝再テスト。Q10 は趣旨が合っていれば 10 点。
