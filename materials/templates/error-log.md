# 誤答ノートの書き方（テンプレート付き）

> 配置先: ドキュメント `00_ガイダンス > 誤答ノートの書き方`
> 受講者は各自 `05_誤答ノート > 氏名` のドキュメントを作成して運用します。

## なぜ誤答ノートか

間違えた問題は、**あなた専用の最強の教材**です。学習科学の研究では、
「思い出す練習（想起練習）」と「間隔を空けた復習（分散学習）」が最も効果の高い
学習法であることが繰り返し確認されています。誤答ノートはこの 2 つを
「自分が実際に間違えた問題」に集中して適用する仕組みです。

- 小テスト直後（その日の夕方）に書く … 記憶が新しいうちに原因を特定
- 翌朝・週次テスト前・修了テスト前に見直す … 間隔を空けた復習で定着

## 運用ルール

1. **毎日**、小テストの採点結果が出たら、間違えた問題をその日のうちに記録する
2. 記録は 1 問 5 分以内。丸写しではなく**自分の言葉で**書く
3. 「なぜ間違えたか」を必ず分類する（下記のエラータイプ）
4. 毎朝の振り返りタイム（9:00〜9:30）に前日分を見直し、
   **答えを隠して自力で解き直す**（読むだけでは効果が薄い）
5. 週次テスト前に、その週の全エントリを解き直す。**2 回連続で正解**した
   エントリには ✅ を付けて卒業させる

### エラータイプ（なぜ間違えたか）

| タイプ | 意味 | 対策 |
|---|---|---|
| 知識不足 | そもそも知らなかった・覚えていなかった | 教材の該当章を読み直し、要点をノートに自分の言葉で書く |
| 混同 | 似た概念・コマンドを取り違えた（例: AD 値とメトリック） | 違いの比較表を自分で作る |
| 読み違い | 問題文・選択肢の読み落とし（「誤っているものを選べ」等） | 設問のキーワードに印を付ける癖をつける |
| ケアレス | 計算ミス・書き間違い | 検算の手順を決める（サブネット計算は 2 通りで確認 等） |

## エントリのテンプレート

ドキュメントに以下をコピーして使ってください。

```
### [Day03-Q4] サブネット計算

- 出題: 192.168.10.0/26 のサブネットで有効ホスト数はいくつか
- 自分の解答: 64 ／ 正答: 62
- エラータイプ: 知識不足（ネットワークアドレスとブロードキャストの2つを引くのを忘れた）
- 正しい理解（自分の言葉で）:
  /26 はホスト部 6 ビット → 2^6 = 64。ただし先頭（ネットワークアドレス）と
  末尾（ブロードキャスト）は PC に割り当てられないので 64 - 2 = 62。
- 解き直し記録: 8/6 ○ ／ 8/7 ○ → ✅ 卒業
```

## 見直しのタイミング（分散学習スケジュール）

| タイミング | やること |
|---|---|
| 当日夕方 | エントリを書く |
| 翌朝 | 答えを隠して解き直し（1 回目） |
| 週次テスト前日 | その週の全エントリを解き直し（2 回目） |
| 修了テスト前（Day 20 午前） | ✅ が付いていないエントリだけを総ざらい |
