# 週次振り返りの書き方（テンプレート付き）

> 配置先: ドキュメント `00_ガイダンス > 週次振り返りの書き方`
> 毎週金曜（Day 5 / 10 / 15 / 20）、週次テストの後に 30 分で行います。
> 記入先: 週次テスト課題のコメント、または `06_振り返り > 氏名` ドキュメント。

## 目的

振り返りは「反省会」ではありません。**来週の学習効率を上げるための作戦会議**です。
テスト結果（事実）→ 原因 → 来週の行動、の順に、短くてよいので毎週書きます。

## テンプレート（コピーして使う）

```
## Week N 振り返り（氏名）

### 1. 数字の確認（事実）
- 週次テスト: __ / 100 点
- 日次小テスト: Day__ __点 / Day__ __点 / Day__ __点 / Day__ __点
- 誤答ノートの新規エントリ数: __ 件（うち ✅ 卒業 __ 件）
- ラボ: 期限内に完了できなかった日: なし / Day__

### 2. ドメイン別の手応え（自己評価 ◎○△×）
- 今週の範囲: ____（例: VLAN ◎ / トランク ○ / STP △ / 無線 ×）

### 3. Keep（うまくいったこと・続けること）
- 例: 朝の解き直しをやったら小テストの点が安定した

### 4. Problem(つまずいたこと・原因)
- 例: STP のポートロールが暗記になっていて、トポロジが変わると解けない
  （原因: 選出の「順番」を手を動かして確認していなかった）

### 5. Try（来週やってみること・具体的に 1〜2 個まで）
- 例: ラボの前に必ずトポロジ図を紙に描いてから始める

### 6. 講師への質問・リクエスト
- 例: OSPF のコスト計算をもう一度解説してほしい
```

## 書き方のコツ

- **Try は最大 2 個**。たくさん書くと全部やらなくなります
- Problem には必ず「原因」を書く。「STP が苦手」で止めず、
  「なぜ苦手のままだったのか」まで掘る
- ×や△が付いたドメインは、翌週月曜朝の弱点解説の対象になります。
  正直に書くほど得をします

## 講師側の使い方

- 全員の「2. ドメイン別の手応え」と週次テストの誤答集計を突き合わせ、
  月曜朝 15 分の弱点解説のテーマを決める
- 「6. 質問」には翌営業日までにコメントで回答する
- Try が実行されたかを翌週の振り返りで一緒に確認する
