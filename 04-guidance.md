# 研修の進め方（受講者向けガイダンス）

> 配置先: ドキュメント `00_ガイダンス > 研修の進め方`
> 初日の朝に必ず通読してください。

CCNA 研修へようこそ。この研修は 20 営業日（約 1 ヶ月・140 時間）で、
ネットワークの基礎から Cisco 機器の設定・セキュリティ・自動化までを、
毎日の「講義 → ラボ → 小テスト」のサイクルで身につけるプログラムです。
準拠資格は **CCNA 200-301 v1.1** です。

## 研修のゴール

- ネットワークの仕組みを自分の言葉で説明できる
- Cisco ルータ・スイッチを CLI で設定・検証・トラブルシュートできる
- **CCNA に合格する**（20 日間の研修修了後、そのまま試験対策フェーズ
  （模試サイクル）に入り、合格まで研修の一部として伴走します。
  自習期間はありません）

## 1 日の流れ（標準時間割）

| 時間 | 内容 | Backlog の課題 |
|---|---|---|
| 9:00〜9:30 | **朝の振り返り**: 前日の解答解説の確認 → 誤答ノートの解き直し → 講義冒頭のウォームアップ想起クイズ | 前日の小テスト課題のコメント |
| 9:30〜13:00 | 講義（教材ドキュメントの学習）。最後に 5 分、白紙に今日の要点を思い出して書き出す | `[DayNN] 講義: …` |
| 13:00〜14:00 | 昼休み | — |
| 14:00〜16:30 | ラボ演習（Packet Tracer） | `[DayNN] ラボ: …` |
| 16:30〜17:30 | 小テスト（30 分）→ 採点後、間違えた問題を**誤答ノート**に記録 | `[DayNN] 小テスト: …` |

あなたの Day 5 / 10 / 15 / 20（各週の締めの日）は小テストの代わりに
**週次テスト**（25 問 / 60 分）です。日付はカレンダーの曜日ではなく、
**自分の研修開始日から数えた日数**で決まります（課題の期限日が基準です）。
最終日 Day 20 は**修了テスト**（60 問 / 120 分）と総合演習ラボを行います。

## Backlog の使い方

### 課題の状態運用

毎日、自分に割り当てられた 3 つの課題（講義 / ラボ / 小テスト）を進めます。

1. 着手したら状態を **「処理中」** に変更
2. 提出物（コメント・添付）を出したら **「処理済み」** に変更
3. 講師が確認すると **「完了」** になります

### 個人情報の扱い（大切なお願い）

- **スクリーンショットを撮る前に**、通知・他のウィンドウ・ブラウザのタブ・
  個人ファイル名を閉じるか隠してください。撮る範囲は指示された画面
  （Packet Tracer・コマンドプロンプト）だけに絞ります。
- **解答コメントに氏名・所属などの個人情報を書かないでください。** 記述式の
  小テストは、採点の一次処理に AI（Anthropic の Claude API）を使うことがあり、
  その際に解答本文が外部 API に送信されます（氏名は送信されません。学習にも
  使われません）。最終的な採点は講師が確認して確定します。
- 週次振り返り（Keep/Problem/Try）は講師のフォロー目的にのみ使い、人事評価には
  流用しません。安心して率直に書いてください。

### 個人情報の扱い（大切なお願い）

- **スクリーンショットを撮る前に**、通知・他のウィンドウ・ブラウザのタブ・
  個人ファイル名を閉じるか隠してください。撮る範囲は指示された画面
  （Packet Tracer・コマンドプロンプト）だけに絞ります。
- **解答コメントに氏名・所属などの個人情報を書かないでください。** 記述式の
  小テストは、採点の一次処理に AI（Anthropic の Claude API）を使うことがあり、
  その際に解答本文が外部 API に送信されます（氏名は送信されません。学習にも
  使われません）。最終的な採点は講師が確認して確定します。
- 週次振り返り（Keep / Problem / Try）は講師のフォロー目的にのみ使い、人事評価には
  流用しません。安心して率直に書いてください。

### 提出のルール

- **講義**: 完了条件のチェックリストを済ませ、疑問点をコメント（なければ「なし」）
- **ラボ**: 完成した `.pkt` ファイル（命名: `dayNN_氏名.pkt`）を課題に添付し、
  検証コマンドの結果と観察レポートをコメント
- **小テスト**: 制限時間内に、コメントへ `Q1: A` の形式で 1 行ずつ解答
  （例は下記）。**教材・検索の参照は禁止**です。実力測定が目的です。

```
Q1: B
Q2: C
...
Q10: スイッチはMACアドレスだけを見て転送するため（記述問題は文章で）
```

### テストの合格基準

| テスト | 基準 | 満たさない場合 |
|---|---|---|
| 日次小テスト（10 問） | 70 点以上 | 翌朝に再テスト |
| 週次テスト（25 問） | 70% 以上 | 週末に補講・再テスト後、次週へ |
| 修了テスト（60 問） | 75% 以上 + 総合演習ラボの要件充足 | 弱点範囲の補講後に再テスト |

点数は評価のためではなく、**弱点を早く見つけて潰すため**にあります。
再テストはペナルティではありません。

## 学習のやり方 — 科学的に効果が確認されている 3 つの習慣

この研修の日課は、学習科学で最も効果が高いとされる方法（242 の研究のメタ分析で
上位 2 位となった「想起練習」と「分散学習」）に沿って設計されています。
読み直すだけの学習は 1 週間後に 34% しか残らないのに対し、思い出す練習をした
場合は 80% 残った、という実験結果もあります。次の 3 つを毎日実行してください。

1. **想起練習（思い出す練習）** — 「読む・眺める」より「思い出す」が記憶に残ります。
   毎朝のウォームアップクイズ、講義末尾の確認問題、講義後の白紙書き出し 5 分、
   夕方の小テストは、すべてこのための仕掛けです。**答えを見る前に必ず自力で
   思い出す**ことを徹底してください。
2. **分散学習（間隔を空けた復習）** — 1 回で覚え切ろうとせず、翌日・3 日後・
   1 週間後と間隔を空けて復習する方が長く残ります。ウォームアップクイズは
   前日・3 日前・1 週間前の範囲から出題されるので、自然に分散復習になります。
3. **誤答ノート** — 間違えた問題はあなた専用の教材です。書き方は
   「誤答ノートの書き方」（00_ガイダンス）を参照。毎日 5 分の記録と翌朝の
   解き直しを続けてください。

週次テスト（Day 5 / 10 / 15 / 20）の後には 30 分の**週次振り返り**
（Keep / Problem / Try）を書きます。書式は「週次振り返りの書き方」
（00_ガイダンス）にあります。

自習用には、通勤時間などに使えるフラッシュカードアプリ（Anki 等）で
ポート番号・AD 値・コマンドを反復するのも効果的です（任意）。

## 質問のルール（重要）

- ラボで **30 分詰まったら必ず質問**してください。粘りすぎは時間の無駄です
- 質問は該当する課題のコメントに「状況 + 試したこと + スクリーンショット」で書く
- 質問すること自体がプラス評価です。遠慮は不要です
