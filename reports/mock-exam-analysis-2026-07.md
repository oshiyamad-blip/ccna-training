# 模試分析レポート — 本試験類似度 と 教材対応度/暗記天井（2026-07）

Opus・high effort で、模試5本×100問＝**計500問を1問ずつ分類**し、(1) CCNA 200-301 v1.1 **本試験仕様**との類似度、(2) 教材と模試の対応度・「暗記だけで満点が取れるか」を判定した。
比較基準は **Cisco 公式の Exam Topics / 試験フォーマット**であり、問題ダンプ・漏洩問題との照合ではない。

---

## 結論（先に3行）

1. **模試は本試験対策として機能する（○）**。ドメイン配点は5本とも公式比率に**完全一致**。ただし①**シミュレーション設問がゼロ**、②**5本が等価フォームでない**の2点が弱点。
2. **教材を丸暗記しても満点は取れない。暗記天井は平均73%**。合格ライン（正答率82〜85%目安）に**暗記だけでは届かない**。
3. 残り27%は「計算・show出力解釈・シナリオ適用（平均24.6%）」＋「教材に記載が無い＝教材の穴（平均2.0%）」。前者はドリルとラボで、後者は教材補強でしか埋まらない。

## Part 1. 模試 vs 本試験の類似度

### 本試験の実仕様（公式ベース・2026年時点）

| 項目 | 本試験 |
|---|---|
| 問題数 | 約100〜120問（Cisco非公開・動的出題で変動） |
| 制限時間 | 120分 |
| 合格点 | 非公開（慣例値 1000点満点中およそ825点。配点重み付けあり＝正答率82.5%の意味ではない） |
| 設問形式 | 単一選択／複数選択／ドラッグ&ドロップ／**シミュレーション・シムレット（実機模擬CLIで設定・検証、通常2〜6問・高配点）**／マルチステップのシナリオ |
| ドメイン比率 | 基礎20 / アクセス20 / IP接続25 / IPサービス10 / セキュリティ15 / 自動化10 |
| 難易度 | 中級。暗記だけでは不十分な実技・シナリオ重視。戻り操作不可。v1.1で生成AI/ML・クラウド管理が自動化ドメインに追加 |

出典: Cisco 公式 Exam Topics（200-301 v1.1 PDF）、Cisco Learning Network、Cisco 公式試験ページ ほか。

### 判定：○（機能する。シミュレーション欠落と5本の不均質が二大課題）

| 観点 | 結果 |
|---|---|
| ドメイン配点比率 | **完全一致（乖離0%）**。5本すべて 20/20/25/10/15/10。この軸は◎ |
| 設問形式 | 単一81.6 / 複数選択11.6 / 対応付け5.0 / 並べ替え1.8（5本平均）。D&Dはテキスト代替済み。**シミュレーション代替が0＝構造的な穴** |
| シナリオ比率 | 平均32%（本試験体感の目標40〜50%にやや不足）。**mock-1が9%と過小**、mock-2が60%と過剰 |
| 難易度 | hard平均8%とやや薄め、easy22%。全体に少し易しめ |
| カバレッジ | ドメイン単位は穴なし。**v1.1新規（生成AI/ML=6.4、クラウド管理=6.7）のサブトピック出題は要現物確認** |
| 5本の均質性 | **最大の弱点**。analyze 6〜33／scenario 9〜60 と回ごとに数倍のブレ＝**非等価フォーム**。GO基準「2回連続85%」の信頼性を損なう |

### 対策としての穴と改善提案

- **【高】シミュレーション/シムレットの代替が皆無**。本試験で最も得点影響が大きい設定・検証技能を測れない。
  → 「show出力＋トポロジ＋設定断片」で複数設問に答えさせる**シムレット模擬を各回3〜5問固定で追加**（対象 2.1 VLAN / 3.4 OSPF / 5.6 ACL / 5.7 L2セキュリティ / 4.1 NAT）。
- **【高】5本が非等価フォーム**。軽い回だけで「2回連続85%」を通過できてしまう。
  → 各回を **シナリオ40±5%・analyze 18〜25%** に収束。または GO基準を「**全5本平均85%かつ各回80%以上**」等へ変更し、軽い回だけの通過を封じる。
- **【中】シナリオ比率が平均32%と不足、mock-1が9%と過小**。→ mock-1を底上げ、hard帯を各回10〜12%へ。
- **【中】v1.1新規論点（6.4/6.7）の出題が本データで未確認**。→ ドメイン6の10問中に最低各1問を固定確保（現物確認）。
- **【低】mock-2は対応付け・並べ替えが0**（D&D露出欠落）／問題数100固定（本試験レンジ下限）。

> 知識カバレッジ・ドメイン配点は確実に機能する（この2軸は◎）。上記【高】2件を補えば◎相当。

## Part 2. 教材と模試の対応度 ／ 暗記天井

### 「教材を暗記したら満点が取れる？」→ 取れない。天井は平均73%

各設問を「**参照Dayの教材本文を丸暗記していれば正答できるか**」で排他分類した（memorizable + application + uncoverable = 100）。

| 模試 | 暗記で解ける | 要・適用（計算/解釈/適用） | 教材の穴（記載なし） | **暗記天井** |
|---|---|---|---|---|
| mock-1 | 84 | 16 | 0 | **84%** |
| mock-2 | 74 | 25 | 1 | **74%** |
| mock-3 | 74 | 26 | 0 | **74%** |
| mock-4 | 68 | 29 | 3 | **68%** |
| mock-5 | 67 | 27 | 6 | **67%** |
| **平均** | **73.4** | **24.6** | **2.0** | **73.4%** |

- **満点は取れない**。最も暗記に優しい mock-1 でも 84%（＝16点は暗記外）。最も厳しい mock-5 は 67%、しかも6%は教材に記載が無く**構造的に取れない**。
- 合格ライン（正答率82〜85%目安）に対し、**暗記天井73%では不合格**。暗記は「土台」であって「ゴール」ではない。

> 補足: この73%は用語想起（recall 平均約18%）だけでなく、「教材の内容理解で解ける（understand）」まで含んだ広義の"教材由来で解ける割合"。狭義の丸暗記だけならさらに低い。

### なぜ満点にならないか（残り27%の内訳）

**(A) 要・適用＝平均24.6%**（暗記では届かない運用力）
- 計算（サブネット/メトリック/ワイルドカード）: mock-5で11%。**ドリル反復**でしか速く正確に解けない。
- show出力解釈・切り分け: mock-2は analyze 33%・シナリオ60%。`show`出力やコンフィグを読む力は**ラボ（ハンズオン）**でしか身につかない。

**(B) 教材の穴＝平均2.0%**（暗記努力と無関係に落とす。**教材側で補強が必要**）

| 追加すべき教材 | Day | 根拠設問 |
|---|---|---|
| config-register / ROMMON / パスワード回復（0x2102, 0x2142） | Day2 | m5 Q5,Q6 |
| `crypto key zeroize rsa`（RSA鍵の再生成/削除） | Day2 | m4 Q7 |
| `login quiet-mode access-class` による管理サブネット除外 | Day16 | m2 Q82, m5 Q83 |
| 隠れ端末問題・RTS/CTS | Day10 | m4 Q35 |
| SWIM / ゴールデンイメージ / 一括アップグレード | Day19 | m4 Q93 |
| YANG（データモデリング言語） | Day19 | m5 Q95 |
| SD-Access ファブリック3ノード役割 | Day19 | m5 Q97 |
| DMVPN / マルチポイントVPN | Day18 | m5 Q90 |

> 最優先は **Day19・Day2・Day16**。特に Day19 は**最頻出（各回Q91-100の10問）なのに教材が浅い**（SWIM/YANG/SD-Access欠落）＝投資対効果が最大の補強ポイント。補強すれば mock-5 の天井は 67%→約73% に回復し、他模試との整合も取れる。

### 教材→模試のカバレッジ（濃淡）

- **厚い**: Day19（自動化, 各回10問で最大の得点源）／Day12（OSPF）／Day15（IPサービス）／Day16（セキュリティ）／Day8（inter-VLAN）
- **薄い**: Day14（NAT, 一貫して2問前後で最薄）／Day6・Day17・Day1・Day4・Day5
- **出題ゼロ**: Day20（総復習・受験前調整回。設問化しないのは**設計通り**で教材欠陥ではない）

### 参照アンカーの整合（軽微・分類の問題）

- OSPF応用（MTU不一致・default-information originate・E1/E2）が模試では **Day12参照だが実体は Day13**（m5 Q56-58）。→ 教材側で **Day12=シングルエリア基礎／Day13=OSPF応用** の範囲境界を明示すると学習者の参照迷子を防げる。
- VLANホッピングのアンカーDayが模試内で二重化（Day7 と Day18）。→ どちらかに寄せるか相互参照を付す。

## Part 3. 合格に向けた示唆 — 「暗記＋ドリル＋ラボ」の三本柱

データが示す学習配分は明快:

1. **暗記だけでは天井73%＝不合格**。recall/understand の73%を固めるのは必要条件。だが合格には**あと10ポイント以上を暗記の外側から積む**。
2. **ドリル（計算）で7〜11%を回収**。サブネット・メトリック・ワイルドカードは最も投資効率の良い上積み（計算ドリル163問が対応）。
3. **ラボ（出力解釈）で apply/analyze を回収**。`show`出力・コンフィグ読解は実機/シミュレータのハンズオン以外に代替なし。本番に近い模試ほどこの比率が上がる。
4. **教材穴（平均2%・最大6%）は制作側で補強**（上表 Day2/16/19 ほか）。

> 本研修は既に「講義（暗記）＋計算ドリル＋Packet Tracer ラボ」の三本柱を備えており、方向性は正しい。上記の**模試へのシムレット追加・フォーム等価化**と**教材8箇所の穴埋め**を行えば、模試の合否予測精度と教材の網羅性がさらに上がる。
