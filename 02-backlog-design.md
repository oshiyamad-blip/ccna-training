# Backlog プロジェクト構成設計

CCNA 研修を Backlog 上で運用するためのプロジェクト設計です。

## 0. マージ先テンプレート（確定）

投入先は、ユーザー提供のプロジェクトテンプレート
**「インフラ開発研修（CCNA）のテンプレート」**（`backlog/project-template.json`）
から作成されるプロジェクト。テンプレートの設定と本設計の適合:

| テンプレート設定 | 本設計との適合 |
|---|---|
| 書式 = **Markdown** | ✅ 教材・課題テンプレート・Wiki投入（画像の `![alt](添付名)` 参照含む）がそのまま使える |
| ドキュメント有効 / Wiki有効（ツリー表示） | ✅ 「Wiki API投入 → ドキュメント移行」の想定どおり |
| 状態 = 未対応/処理中/処理済み/完了 | ✅ 本設計の状態運用と一致 |
| 子課題あり（孫課題なし） | ✅ 受講者別プロジェクト方式なら子課題は補助的に使用 |
| ファイル共有有効 / Git無効 | ✅ .pkt 添付は課題添付とファイル共有で対応 |
| 種別 = 基本操作/プロジェクト設定/要望/その他 | ⚠️ 研修用の種別（講義/ラボ/小テスト/週次テスト/模試）は投入スクリプトが**追加**する。テンプレート由来の4種別は残るので、不要なら手動整理するか、テンプレート側に研修用種別を追加しておくと新規プロジェクトが最初から揃う（推奨） |
| カテゴリー/マイルストーン/カスタム属性 = 空 | ✅ 投入スクリプトが作成する |

## 1. プロジェクト基本設定

| 項目 | 設定 |
|---|---|
| プロジェクト名 | CCNA 研修 2026 |
| プロジェクトキー | `CCNA` |
| チャート | ガントチャート・バーンダウンを有効化（進捗の可視化に使用） |
| 参加者 | 受講者 = 一般ユーザー、講師 / メンター = プロジェクト管理者 |

## 2. 種別（課題タイプ）

| 種別 | 用途 | 色の目安 |
|---|---|---|
| `講義` | その日のドキュメント学習。教材ドキュメントへのリンクと学習目標を記載 | 緑 |
| `ラボ` | Packet Tracer 演習。手順書リンクと `.pka` / `.pkt` ファイルを添付 | 青 |
| `小テスト` | 日次 10 問。設問は課題本文、解答はコメントで提出 | 赤 |
| `週次テスト` | Day 5/10/15/20 の 25 問（修了テストは 60 問） | 橙 |

## 3. カテゴリー（= 試験ドメイン）

1. `1-ネットワーク基礎`
2. `2-ネットワークアクセス`
3. `3-IPコネクティビティ`
4. `4-IPサービス`
5. `5-セキュリティ基礎`
6. `6-自動化とプログラマビリティ`

課題に付与しておくと「どのドメインでつまずいているか」をカテゴリー別の絞り込みで分析できる。

## 4. マイルストーン（発生バージョン）

| マイルストーン | 期間 | 内容 |
|---|---|---|
| `Week1-ネットワーク基礎` | Day 1〜5 | ドメイン 1 |
| `Week2-ネットワークアクセス` | Day 6〜10 | ドメイン 2 |
| `Week3-IPコネクティビティとIPサービス` | Day 11〜15 | ドメイン 3・4 |
| `Week4-セキュリティと自動化` | Day 16〜20 | ドメイン 5・6 + 総まとめ |

週次テスト合格（70% 以上）をもってマイルストーン完了とする。

## 5. 課題の構成

Week0（5 日 × 3 = 15）+ 本編（20 日 × 3 = 60）+ 試験対策フェーズ（8）で
**計 83 課題**（`--skip-precourse` 時は Week0 を除き Day00 を追加して 69 課題）。命名規則:

```
[Day01] 講義: ネットワークの全体像と OSI / TCP-IP モデル
[Day01] ラボ: PT入門 — 最小構成の作成とカプセル化の観察
[Day01] 小テスト: OSI モデルとカプセル化
```

- **担当者**: 受講者本人（複数受講者の場合は受講者ごとに課題を複製するか、
  子課題で受講者別に分ける）
- **期限日**: 研修開始日から営業日ベースで自動計算（投入スクリプトが設定）
- **状態運用**: 未対応 → 処理中（着手）→ 処理済み（提出）→ 完了（講師確認後）

課題の一括投入は [`scripts/create-backlog-issues.mjs`](./scripts/create-backlog-issues.mjs) を使用。

### 課題テンプレート（種別ごとに登録）

**講義:**

```
## 学習目標
- （このセッションで説明できるようになること）

## 教材
- ドキュメント: （教材ドキュメントへのリンク）

## 進め方
1. 教材ドキュメントを通読する（3.5h 目安）
2. 理解チェック（教材末尾の確認問題）を解く
3. 疑問点はこの課題のコメントに書き残す → 講師が回答

## 完了条件
- [ ] 教材を通読した
- [ ] 確認問題を解いた
- [ ] 疑問点をコメントした（なければ「なし」とコメント）
```

**ラボ:**

```
## ゴール
- （構築するトポロジと動作要件）

## 手順書
- ドキュメント: （ラボ手順書へのリンク）
- 添付: 開始用ファイル（.pkt / .pka）

## 提出物
- 完成した .pkt ファイルをこの課題に添付
- 検証コマンドの実行結果（スクリーンショットまたはテキスト）をコメント

## 完了条件
- [ ] 要件どおりの疎通 / 動作を確認した
- [ ] 完成ファイルを添付した
```

**小テスト:**

```
## ルール
- 制限時間 30 分・教材参照なしで解くこと
- 解答はこの課題の**コメント**に「Q1: A」の形式で提出
- 正答率 70% 未満の場合は翌朝に再テスト

## 設問
（10 問をここに記載。選択式中心 + 記述 1〜2 問）

## 解答・解説
- 翌日 9:00 に「解答解説」ドキュメントのリンクをコメントで公開
```

## 6. ドキュメント機能のフォルダ構成

```
📁 00_ガイダンス
   ├─ 研修の進め方（このプロジェクトの使い方）
   ├─ カリキュラム全体表
   └─ Packet Tracer 導入マニュアル
📁 01_教材
   ├─ 📁 Week1_ネットワーク基礎
   │    ├─ Day01_ネットワークの全体像とOSIモデル
   │    ├─ Day02_Cisco IOS の基本操作
   │    └─ ...
   ├─ 📁 Week2_ネットワークアクセス
   ├─ 📁 Week3_IPコネクティビティとIPサービス
   └─ 📁 Week4_セキュリティと自動化
📁 02_ラボ手順書
   ├─ 📁 Week1 … Day01_PT入門ラボ 手順書 など
   └─ ...
📁 03_小テスト解答解説
   ├─ Day01_解答解説（翌日公開）
   └─ ...
📁 04_講師用（権限で受講者から隠す）
   ├─ 採点基準
   └─ 週次テスト・修了テスト原本
```

- 解答解説と講師用フォルダは**公開タイミング / 権限に注意**（ドキュメント機能の
  権限管理で受講者の閲覧を制御）。
- **投入方法**: ドキュメント API は未提供のため、(a) Markdown 教材を Wiki API で
  一括投入 → Wiki 移行機能でドキュメント化、または (b) 手動貼り付け。

## 7. 小テストの運用フロー

1. 小テスト課題は投入時から本文に設問を記載（期限日 = 当日）
2. 受講者はコメントで解答を提出し、状態を「処理済み」に変更
3. 講師（または自己採点）が採点し、点数をコメント → 状態「完了」
4. 翌朝、解答解説ドキュメントのリンクをコメントで公開
5. 70% 未満の受講者には「再テスト」課題を追加（テンプレートから作成）

> 将来の拡張: Backlog API の Webhook + 採点 Bot（コメントの解答をパースして
> 自動採点・自動コメント）を作れば採点を自動化できる。まずは手動運用で開始し、
> 受講者数が増えたら Bot 化を検討する。

## 8. 進捗の可視化

- **ボード**: 状態別（未対応 / 処理中 / 処理済み / 完了）で当日のタスクを確認
- **ガントチャート**: 20 日間の全体進捗と遅れの検知
- **カテゴリー絞り込み**: ドメイン別の得意 / 不得意の把握
- **週次テスト結果**: マイルストーンの完了状況で判定
