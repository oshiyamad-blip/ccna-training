# Cisco Packet Tracer 導入・操作マニュアル（受講者配布用）

本研修のラボ演習では、Cisco 公式の無料ネットワークシミュレータ
**Cisco Packet Tracer** を使用します。実機がなくてもルータ・スイッチの設定操作を
本物同様の CLI で練習できます。

> 対象バージョン: Packet Tracer 9.x（8.2 以降であれば演習ファイルは動作します）

---

## 1. インストール（初日までに完了させてください）

### 1-1. Cisco NetAcad（Skills for All）アカウントの作成

Packet Tracer のダウンロードには無料アカウントが必要です。

1. <https://www.netacad.com/cisco-packet-tracer> にアクセス
2. 「Sign up」から無料アカウントを作成（メールアドレスのみで可。Google アカウントでも登録可能）
3. 登録したメールアドレスの確認リンクをクリックして有効化

> **注意**: Packet Tracer のインストーラを受講者間でコピーして配布することは
> ライセンス上できません。必ず各自のアカウントでダウンロードしてください。

### 1-2. ダウンロードとインストール

1. ログイン後、Resource Hub（<https://www.netacad.com/resources/lab-downloads>）から
   自分の OS 用のインストーラをダウンロード
   - Windows: 64bit `.exe`
   - macOS: `.dmg`（Apple Silicon / Intel 共通）
   - Linux: Ubuntu 向け `.deb`
2. インストーラを実行し、既定設定のままインストール
3. 初回起動時に NetAcad アカウントでのログインを求められるので、1-1 のアカウントでログイン
   （「Keep me logged in」にチェックを入れると毎回のログインを省略できます）

### 1-3. 動作確認

1. Packet Tracer を起動し、新規ファイルが開くこと
2. 画面下部のデバイス一覧からルータ（例: ISR 4331）を 1 台ワークスペースに配置できること
3. 配置したルータをクリックし「CLI」タブでプロンプト（`Router>`）が表示されること

ここまで確認できたらセットアップ完了です。うまくいかない場合は
Backlog の課題「[Day00] 環境構築」にコメントで質問してください。

## 2. 画面と基本操作

| 部位 | 役割 |
|---|---|
| 中央の白い領域（ワークスペース） | 機器を配置してトポロジを作る場所 |
| 左下のデバイスボックス | ルータ / スイッチ / PC / 無線機器などの選択 |
| 接続（稲妻アイコン） | ケーブルの選択。迷ったら「自動選択（Automatically Choose Connection Type）」 |
| 右側ツールバー | 選択 / 移動 / 削除 / インスペクト |
| 右下の「Realtime / Simulation」 | 通常動作とシミュレーション（パケット可視化）の切り替え |

### よく使う操作

- **機器の配置**: デバイスボックスから機器をワークスペースへドラッグ
- **ケーブル接続**: 接続アイコン → ケーブル種類を選択 → 機器 1 をクリックしポートを選択 → 機器 2 も同様
  - PC ↔ スイッチ、スイッチ ↔ ルータ: ストレート（実線アイコン）
  - 同種機器同士: クロス（点線アイコン）※最近の機種は Auto-MDIX でどちらでも可
- **リンク状態**: ケーブル両端の● が**緑 = リンクアップ**、**赤 = ダウン**、**橙 = STP ブロック中**
- **保存**: `File > Save As` で `.pkt` 形式。**課題の指示どおりのファイル名**
  （例: `day03_subnet_yamada.pkt`）で保存し、Backlog の課題に添付して提出

### 機器の設定方法（2 通り）

1. **CLI タブ（推奨・研修の基本）**: 機器をクリック → 「CLI」タブ。実機と同じ IOS コマンドを入力
2. Config タブ: GUI で設定できるが、**本研修では CLI 操作の習得が目的のため使用禁止**
   （出力される設定の確認に見る程度は可）

### PC の設定

PC をクリック → 「Desktop」タブ:

- **IP Configuration**: IP アドレス / サブネットマスク / デフォルトゲートウェイの設定
- **Command Prompt**: `ping` / `tracert` / `ipconfig` による疎通確認

## 3. シミュレーションモード（パケットの可視化）

講義で学んだカプセル化や ARP、STP の動きを「目で見る」ための機能です。

1. 右下の「Simulation」をクリック
2. 「Show All/None」→「Edit Filters」で観察したいプロトコル（例: ICMP、ARP）だけにフィルタ
3. PC から ping を実行し、「再生（▶）」ボタンでパケットの移動をコマ送りで観察
4. 封筒アイコンをクリックすると、**各層のヘッダの中身**（OSI Model / PDU 詳細）を確認できる

---

## 4. ラボ課題ファイル（.pka / .pkt）の使い方

- **.pkt**: 通常のトポロジ保存ファイル。開いて自由に操作 → 完成品を保存して提出
- **.pka**（アクティビティファイル）: 課題内蔵ファイル。開くと指示ウィンドウと
  **採点機能（Check Results）** が付いており、完成度が % で表示される
  - 「Check Results」→「Assessment Items」で未達成項目を確認できる
  - 完了条件: 原則 **100%**（課題ごとの指示に従うこと）
  - 提出時は完了スコアが見えるスクリーンショットを Backlog 課題のコメントに貼る

## 5. トラブルシューティング

| 症状 | 対処 |
|---|---|
| インストーラがダウンロードできない | NetAcad にログインしているか確認。会社プロキシ経由の場合は情シスに確認 |
| 起動時のログインに失敗する | パスワード再設定。オフライン環境では「Guest Login」も可（保存回数に制限あり） |
| リンクが赤いまま | ケーブル種類の間違い / ポートの shutdown を確認（`no shutdown`）。機器の電源（物理タブ）も確認 |
| CLI に何も表示されない | Enter キーを 1〜2 回押す。起動直後は IOS ブート完了まで数十秒待つ |
| 設定が消えた | `copy running-config startup-config`（または `write memory`）で保存したか確認。PT ファイル自体の保存（.pkt）も忘れずに |
| 日本語入力が文字化けする | hostname やバナーには半角英数字のみ使用する |

## 6. 研修での運用ルール

1. ラボはすべて Backlog の「ラボ」課題の手順書（ドキュメント）に沿って実施する
2. 完成ファイルは命名規則 `dayNN_テーマ_氏名.pkt` で課題に添付する
3. 検証コマンド（`show ip route` 等）の結果をコメントに貼る
4. 30 分詰まったら自力解決をやめ、課題コメントで質問する（質問も評価対象。遠慮しない）
5. 授業時間外の自主練習も推奨。Packet Tracer は個人利用無料なので自宅 PC にも導入可
